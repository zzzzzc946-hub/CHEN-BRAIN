# MAX选题库调试记录

## 文件定位

本文件用于记录 MAX 选题库在真实使用前的调试过程。

它不是外部情报日报。

它不是选题库本体。

它不是内容生产 Skill。

它记录的是：

```text
一条外部情报 / 候选材料 / 选题卡片，
在给 MAX 口喷之前，
经过 CHEN / MAX / AI 讨论后，
具体调试了什么，
为什么要调试，
调试后变成了什么样，
以及这次调试应当反向更新哪些规则。
```

本文件的主作用是：

```text
校准 MAX选题库里“给 MAX 口喷的材料卡片”应该长什么样。
```

同时，它也会反向影响：

```text
外部情报日报怎么收集
日报里的“初步 MAX 可讲视角”怎么写
关键词与来源池应该补充哪些检索方向
选题规则应该如何更新
```

一句话：

```text
这里记录的不是素材本身，而是素材被调试成 MAX 可讲材料的过程。
```

---

# 一、记录原则

每一次调试，都必须记录清楚四件事：

```text
1. 原来是什么
2. 哪里不对
3. 怎么调对
4. 调试后要更新哪些规则
```

调试记录不能只写结论。

必须保留调试过程。

因为后续系统会越来越准，靠的不是一次性写对，而是反复记录：

```text
AI 最初怎么理解
CHEN / MAX 怎么纠偏
纠偏后产生了什么更高质量的判断
这个判断是否要沉淀成长期规则
```

# 2026-06-16｜约会香水案例的切口能量调试

## 1. 调试对象

约会香水案例。

这条材料原本来自 CHEN 对具体约会细节的构造：

```text
一个男生约会前精心准备了穿搭、餐厅、展示面，甚至想用香水增加精致感。
但他平时很少真正使用香水，也不懂品牌、浓度、场合和用量。
结果约会时香水味太冲，或者女生问他喷的什么，他一说出来是杂牌、盗版、廉价网红香。
前面所有准备瞬间破掉。
```

## 2. 原始材料

原始判断是：

```text
香水本来是加分项，但如果使用不自然，可能变成身份破绽。
懂的女生会判断：这个人不是长期有这种生活习惯，而是在临时补一个“精致男人”的人设。
```

这个素材的价值在于：

```text
它是一个极小、具体、可感的约会场景。
它能反映价值感、生活方式熟悉度、审美细节和身份识别。
```

## 3. AI 初始处理

AI 初始给出的推荐切口是：

```text
你用的香水，暴露的不是价格，是你根本不熟这种生活。
```

AI 当时的理解偏向：

```text
香水是一个细节破绽。
错误香水会暴露生活方式不熟。
展示面可能被一个小细节打穿。
```

## 4. CHEN / MAX 反馈

MAX 反馈：

```text
这个切入点已经有 70 分，可以讲。
但“香水暴露你不熟这种生活”这个视角稍微偏负，甚至有点偏穷人视角。
```

MAX 更认可的方向是：

```text
不要从指责、审判、羞辱进入。
更好的方式是像 JZ 一样，提供知识、品味、生活方式和选择建议。
```

MAX 提出的更好方向：

```text
这更像是“约会香水选择”。
一个细节就可以让女生快速上头。
这个角度能量更好。
```

CHEN 补充判断：

```text
重点不是一定要讲“你被看穿了”。
重点是这个具体香水场景能引发 MAX 的观点。
这个素材搜集就是成功的，因为它触发出了更准确的内容方向。
```

## 5. 问题诊断

这次偏差属于：

```text
切口能量不对。
问题诊断成立，但表达容易偏负。
容易从“价值提供”滑向“细节审判”。
容易像传统情感号一样制造压迫感。
```

具体问题：

```text
“你根本不熟这种生活”这句话虽然锋利，但容易给人居高临下、批判、羞辱的感觉。
它更像在指出用户的低位，而不是带用户进入一个更好的选择。
```

MAX 更想要的表达气质：

```text
没有戾气。
不居高临下。
不批判穷人。
不压迫用户。
像 JZ 一样，用知识科普、品味判断和生活方式启发去提供价值。
```

## 6. 调试后的新方向

调试后，这条材料不应主要讲：

```text
你喷错香水，会暴露你不熟这种生活。
```

而应更适合讲：

```text
约会香水选择，是一个能让女生快速感受到品味、干净感、距离感和生活细节的入口。
```

它的方向从：

```text
负向破绽识别
```

调整为：

```text
正向品味建议 / 约会细节价值提供 / 生活方式启发
```

## 7. 调试后的可用表达

可用表达 1：

```text
约会时，一个香水细节就能让女生快速感受到你的品味。
```

可用表达 2：

```text
约会香水不是为了让人闻到你喷了香水，而是让她觉得你这个人很干净、很有距离感。
```

可用表达 3：

```text
真正加分的香水，不是贵，而是符合你的气质、场合和距离。
```

可用表达 4：

```text
一个男人的精致感，不是靠香水堆出来的，而是靠他知道什么场合该用什么味道。
```

保留为底层判断的表达：

```text
你用的香水，暴露的不是价格，是你是否真的熟悉这种生活。
```

注意：

```text
这句话可以作为底层判断，但不一定适合作为最终外部切口。
```

## 8. 沉淀规则

本次调试沉淀出以下规则：

```text
外部情报可以从翻车、尴尬、破绽进入。
但 MAX 最终表达不能停留在羞辱、批判、低位审判。
```

进一步规则：

```text
抓取可以抓破绽。
日报可以记录破绽。
选题库给 MAX 的材料，必须尽量转成正向价值、知识启发、品味判断和场景建议。
```

切口优先级：

```text
正向价值提供
↓
知识 / 品味 / 生活方式启发
↓
具体场景建议
↓
问题诊断
↓
刺痛 / 羞辱 / 批判
```

这条规则尤其适用于：

```text
约会细节
展示面破绽
审美误判
生活方式不熟
情感关系低位
```

## 9. 需要迭代的文档

需要同步更新：

```text
02-关键词与来源池-V0.1
```

更新方向：

```text
在关键词池里补充正向价值提供类关键词。
例如：
约会香水选择
男士约会香水
女生喜欢的男香
香水用量 场合
干净感 香水
男士香水 距离感
```

需要同步更新：

```text
03-MAX外部情报日报模板-V0.1
```

更新方向：

```text
在“初步 MAX 可讲视角”中补充规则：
可以记录问题诊断，但应优先尝试转成正向价值提供、知识科普、品味判断和场景建议。
避免停留在羞辱、批判和低位审判。
```

后续需要同步更新：

```text
选题规则
```

更新方向：

```text
新增“切口能量调试规则”：
给 MAX 口喷的材料卡片，不能只指出用户的问题；
更应该提供一个能让 MAX 展开知识、品味、生活方式和价值判断的正向入口。
```

# 2026-07-14｜展示面同质素材批量去重与选题库收缩

## 1. 调试对象

当前对话中连续处理的展示面、生活化拍摄、朋友圈排行榜、物质表达、审美与第一判断类外部素材。

## 2. 原始材料

本轮共复核 18 条唯一外部素材，包含：

```text
生活化展示面
活人感
普通场景
优势展示
物质表达
单一标签
五张照片模板
审美积累
老钱风
展示面定义
价值翻译
九宫格排版
朋友圈内容排行榜
朋友圈第一判断
搭讪与展示面效率
一次性高消费
```

其中还出现：

```text
同一作品不同分享链接
同一逐字稿重复提交
多个作品围绕同一判断反复表达
排行榜型内容高度同质
```

## 3. AI 初始处理

AI 初始处理倾向于：

```text
每收到一条素材，就单独生成一条 04 日报和一张 07 候选卡片。
只要与展示面相关、数据不差，就判断为“可入库候选”。
```

这种处理在单条判断时看似成立，但批量横向比较后出现明显问题：

```text
生活化、活人感、低门槛、普通场景、一次性高消费被拆成多张近似卡片。
多个“朋友圈从夯到拉”排行榜重复进入候选。
同一视频更换分享链接后被再次处理。
07 逐渐变成外部素材列表，而不是 MAX 独立判断库。
```

## 4. CHEN / MAX 反馈

CHEN 要求：

```text
把此前所有生产的日报统一迭代。
该入库的入库。
该迭代的迭代。
一次性写入文档。
```

这意味着本轮不能继续按单条素材局部判断，而必须做一次全局去重、聚类和收缩。

## 5. 问题诊断

本次偏差属于：

```text
把 KEEP 误当成必须入 07。
把来源单位误当成选题单位。
缺少跨素材横向去重。
高热度替代了独立判断判断。
重复素材只做了链接去重，没有做语义去重。
选题库数量膨胀，降低 MAX 口喷效率。
```

核心问题：

```text
04 记录“外部世界说了什么”。
07 应记录“MAX 最终能讲出什么”。
二者单位不同，不能一对一映射。
```

## 6. 调试后的新方向

本轮改为：

```text
18 条唯一素材全部进入 04 的 KEEP / BUFFER 体系。
同一作品的不同链接合并并更新热度。
同质排行榜内容保留为 BUFFER 或辅助来源。
跨来源聚类后，只形成 9 张独立可口喷卡片。
```

新的处理关系：

```text
来源层：允许多。
判断层：必须少。
卡片层：一个核心判断只保留一张。
```

## 7. 调试后的可用表达

```text
KEEP 代表值得留在日报，不代表必须进入选题库。
```

```text
04 的单位是外部素材，07 的单位是 MAX 独立判断。
```

```text
不同作品可以共同支撑一张卡片，不需要每条来源都长出一个选题。
```

```text
热度可以证明用户在意，但不能证明它是一个新选题。
```

```text
来源越多，越需要收缩判断，而不是膨胀选题库。
```

## 8. 沉淀规则

本轮沉淀以下长期规则：

```text
1. 同一作品不同分享链接，只更新来源和热度。
2. 不同作品表达同一判断，04 分开保留，07 合并。
3. 排行榜、清单和泛技巧内容，只有产生新机制时才独立入库。
4. 每轮结束必须横向比较所有候选，不允许逐条孤立入库。
5. 07 每张卡片必须回答：它比现有卡片新增了什么？
6. 无新增用户痛点、场景、机制或可执行判断时，转为辅助来源或 BUFFER。
```

## 9. 需要迭代的文档

```text
05-选题规则：升级到 V2.4，增加三层去重、N:1 合并和 07 限流规则。
04-MAX外部情报日报：写入 18 条去重后的 KEEP / BUFFER 素材。
07-MAX选题库：只写入 9 张聚类后的独立判断卡片。
```
