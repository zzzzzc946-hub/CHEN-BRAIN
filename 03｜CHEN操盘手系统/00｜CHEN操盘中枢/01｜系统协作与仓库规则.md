# 01｜系统协作与仓库规则

## 一、文档定位

本文档是 CHEN BRAIN 当前的系统协作、仓库维护、AI 执行规则入口。

本仓库是 Obsidian Markdown 知识库，不是普通代码项目。

所有动作最终服务：

```text
更懂 MAX
↓
更懂 MAX 客户群体
↓
判断是否有价值、是否有效
↓
产出更准的选题和内容
```

## 二、硬规则

1. 本仓库是 Obsidian Markdown 知识库，不是代码项目。
2. Codex 不得主动创建 `src`、`app`、`package.json` 等代码项目结构。
3. Codex 不得修改或移动 `02｜MAX顶层思维系统`。
4. Codex 不得主动扩系统。
5. Codex 可以在用户明确要求时 commit / push。
6. 不恢复 `00｜MAX IP项目总控台`。
7. 不恢复 `04｜MAX × CHEN AI协同系统`。
8. 不创建新的一级目录，除非 CHEN 明确要求。

## 三、新语料归属判断

新语料进入时先判断归属：

```text
MAX思想源
→ 02｜MAX顶层思维系统

客户、选题、内容、纠错、发布反馈
→ 03｜CHEN操盘手系统

仓库规则、协作规则、阶段记录
→ 03｜CHEN操盘手系统/00｜CHEN操盘手系统总纲
```

## 四、ChatGPT / Codex / Obsidian 分工

ChatGPT：

- 理解 MAX
- 理解 MAX 客户群体
- 提炼语料增量
- 产出选题和内容
- 进行内容判断和迭代

Codex：

- 文件整理
- 仓库维护
- 文档合并
- 结构迁移
- 工具和自动化执行
- 用户明确要求时 commit / push

CHEN：

- 最终判断
- IP 操盘
- 关键取舍
- 内容方向确认

Obsidian：

- 本地知识库
- 长期沉淀
- 可检索结构

## 五、GitHub 与 Obsidian

GitHub 是远端同步和 ChatGPT 读取仓库的云端入口。

Obsidian 是本地知识库和长期沉淀入口。

Codex 可以维护仓库文件、执行迁移、做 Git 状态检查。

需要提交或推送时，必须由 CHEN 明确要求。

## 六、AI 协作纠偏

AI 协作不是独立一级系统，而是 CHEN BRAIN 的工作方式。

重复偏差可以沉淀，但不因此扩系统。

重点记录：

- 重复性偏差
- 结构性误判
- 高价值复用经验
- 用户明确修正后的长期规则

纠偏只记录会影响 MAX IP 主线推进的偏差与经验。
