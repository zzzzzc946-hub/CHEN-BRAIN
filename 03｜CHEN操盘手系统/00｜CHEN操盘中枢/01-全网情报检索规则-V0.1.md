# 全网情报检索规则 V0.1

## 文件定位

本文件属于 CHEN 操盘侧的外部情报系统。

它是给 AI 读取的情报判断规则，不是技术执行方案。

本文件只回答：

> AI 应该从全网信息中识别什么、筛选什么、保留什么，并整理进《MAX外部情报日报》。

本文件不负责：

- 写稿
- 生成标题
- 生成最终选题
- 替 MAX 做最终内容判断
- 规定具体爬虫技术
- 规定各平台采集方式
- 规定 Agent 字段协议

后续需要单独建立：

- 《关键词与来源池》：动态维护关键词、账号、平台来源和有效/无效检索词。
- 《MAX外部情报日报》：承接每日或每轮情报输出。
- 《平台采集执行方案》：后续再写，专门规定抖音、小红书、视频号、微博等平台如何采集。

---

# 一、系统定位

全网情报检索系统不是热点搬运系统。

不是观点收集系统。

不是同行内容搬运系统。

不是爬虫系统本身。

它的本质是：

> 为 MAX 从外部世界中发现可被重新解释的现实信号。

这些现实信号必须最终服务 MAX 当前阶段主线：

> 真实价值如何被表达，如何形成价值感，如何制造吸引力，并最终影响关系结果。

外部信息不是因为“热”才值得抓。

外部信息值得抓，是因为它能承载 MAX 对以下问题的判断：

- 价值感
- 价值表达
- 身份表达
- 审美判断
- 阶层识别
- 场景熟悉度
- 情感反馈
- 关系结果
- 高端展示面
- 用户为什么没有被正确看见

一句话：

> MAX 不搬运观点，MAX 借现实事件讲自己的判断。

# 二、系统输入与输出

## 1. 输入是什么

本系统的输入是全网外部信息，包括：

- 短视频
- 图文笔记
- 微博热点
- 评论区
- 直播间连麦
- 情感投稿/匿名案例
- 知乎问答
- B站长视频
- 微信公众号文章
- 明星/公众人物事件
- 富人阶层/名利场事件
- 品牌活动
- 体育赛事
- 影视片段
- 用户吐槽
- 同行内容
- 社会现象
- 生活方式与审美素材
- 时尚、穿搭、旅行、拍摄、设计素材

## 2. 输出是什么

本系统的唯一正式输出是：

> 《MAX外部情报日报》

日报回答：

> 外部世界发生了什么，里面有没有值得 MAX 讲的现实素材。

## 3. 输出不是什么

日报不是选题库。

日报不是标题库。

日报不是口播稿。

日报不是最终内容判断。

日报只负责把外部信息压缩成 CHEN 可以快速判断的情报材料。

后续是否进入《MAX选题库》，必须经过《选题规则》二次筛选。

# 三、抓取边界

全网情报只抓与 MAX 当前主线有关的外部现实信号。

优先抓以下方向。

## 1. 价值感破绽

一个男人想表达价值，但细节暴露出不熟、不稳、不自然。

例如：

- 高端餐厅不熟
- 酒店拍照太像打卡
- 穿搭全是 logo
- 香水喷太重
- 点菜先看价格
- 礼物送得太用力
- 朋友圈太像临时包装

核心判断：

> 他不是没有准备，而是准备得太像临时进入这个世界。

## 2. 极小动作暴露身份

一个小动作，让女生或外界完成身份判断。

例如：

- 问服务员时怯场
- 付款时犹豫
- 红酒不会拿杯
- 不会使用高端场景里的物品
- 在不该拍照的地方一直拍
- 对高端服务表现得过度兴奋或紧张

核心判断：

> 真正懂的人，不看你有没有摆出符号，而看你和这个场景是否自然匹配。

## 3. 情感关系中的价值表达

男人现实条件不错，但在关系中表达成低位、供养、焦虑或讨好。

例如：

- 用付出换确定性
- 用请客送礼证明自己
- 女生冷淡后反复加码
- 条件很好但只被当供养者
- “人不错，但没感觉”

核心判断：

> 不是没有价值，而是价值被表达成了需要感。

## 4. 具体情感关系事件

两性关系、家庭关系、原生家庭、人格模式或相处问题中出现的具体事件。

例如：

- 直播间连麦中的真实情感案例
- 女生吐槽一次约会里的具体细节
- 男生因为焦虑反复确认关系
- 原生家庭导致的讨好、回避、控制或低边界
- 条件不错的男人在相处中被识别成供养者
- 关系里某个具体动作引发冷淡、误解、压迫或逃离

核心判断：

> 情感素材不是用来讲情感技巧，而是用来观察一个人的价值表达、边界、主体性和关系反馈。

## 5. 阶层与名利场规则

通过富人、家族、名流、品牌、体育、影视等事件，观察身份和价值感如何被识别。

例如：

- VIP 活动为什么不再拼预算
- 富人旅行如何体现阶层差异
- 世界杯包厢如何区分圈层
- 富豪婚礼如何展示家族资源
- 电影片段中的身份压迫和场域权力
- 品牌晚宴、时装周、F1、电影节中的座位、邀请、权限和场景差异

核心判断：

> 高级不是花了多少钱，而是进入了什么场、懂不懂那个场。

## 6. 审美与生活方式误判

一个人以为自己在表达高级，实际暴露了审美粗糙。

例如：

- 老钱风变成角色扮演
- 新贵穿搭过度用力
- 高端场景被拍成游客照
- 有钱但没品
- 消费很贵但整体很土
- 把正确单品堆在身上，但整体不像那类人

核心判断：

> 价值感不是把贵的东西摆出来，而是让它看起来本来就属于你。

## 7. 公共热点里的 MAX 视角

热点、明星、绯闻、体育赛事可以抓，但必须有 MAX 重解释空间。

例如：

- 明星恋情中的身份差异
- 富豪家族的择偶逻辑
- 体育赛事里的圈层座位和社交规则
- 公众人物翻车暴露审美或身份错位
- 某个公共事件暴露男性价值表达方式的变化

核心判断：

> 不追热点本身，追热点背后的社会判断。

# 四、平台优先级

不同平台不是同一种用法。

V0 阶段按两层优先级看。

## 第一优先级：短内容与高频反馈平台

这些平台适合高频发现事件、现象、动作、吐槽和评论区共识。

### 1. 小红书

重点看：

- 女性视角
- 评论区真实语言
- 展示面评价
- 男生朋友圈吐槽
- 审美误判
- 生活方式细节
- 约会场景反馈

小红书适合找：

> 女生如何判断一个男人的照片、朋友圈、穿搭、约会表现和生活方式。

### 2. 抖音

重点看：

- 爆款表达
- 同行内容
- 情感痛点
- 社会现象
- 阶层与审美讨论
- 热点事件延伸

抖音适合找：

> 哪些现实判断正在被大量传播，哪些话题具有公共情绪。

### 3. 视频号

重点看：

- 中年男性
- 情感关系
- 家庭关系
- 供养者模式
- 现实判断
- 熟人社会里的价值消耗

视频号适合找：

> 更现实、更传统、更家庭化、更中年化的男性处境。

### 4. 微博

重点看：

- 热点
- 明星绯闻
- 社会情绪
- 公共事件
- 评论区共识
- 快速发酵的争议

微博适合找：

> 公共事件里的身份判断、舆论反馈和社会共识。

## 第二优先级：深内容与案例沉淀平台

这些平台适合补充长案例、深解释、人物故事、生活方式材料和社会规律。

### 5. B站

重点看：

- 长视频分析
- 影视片段
- 穿搭/审美/设计
- 生活方式
- 情感案例
- 阶层、名利场和文化评论

B站适合找：

> 能被拆成具体场景、具体动作和长期规律的深内容素材。

### 6. 知乎

重点看：

- 真实故事
- 匿名经历
- 两性关系问题
- 原生家庭
- 择偶观
- 阶层讨论
- 社会规律

知乎适合找：

> 具体经历、关系问题和社会判断中的高质量故事材料。

注意：

> 知乎匿名故事可以作为典型场景使用，不默认当成已验证事实。

### 7. 微信公众号

重点看：

- 深度文章
- 品牌活动
- 名利场观察
- 人物故事
- 生活方式趋势
- 时尚、设计、旅行、商业与阶层材料

微信公众号适合找：

> 比短视频更完整的背景、案例和趋势解释。

后续可扩展：

- YouTube
- Instagram
- X
- 影视片段素材库
- 奢侈品/名利场新闻源

这些平台的具体采集方式，不写在本文档里，后续进入《平台采集执行方案》。

# 五、信息采集对象与最低可用标准

AI 识别情报时，不要求每条素材都拿到完整逐字稿。

短视频、视频号、抖音、小红书等平台，很多时候无法稳定采集完整正文或逐字稿。

所以本系统按“信息完整度”判断，而不是按“是否有全文”判断。

## 1. 必须尽量采集的信息

一条候选情报，至少要尽量包含：

- 平台
- 链接
- 作者/账号
- 标题或封面标题
- 正文/文案/简介
- 发布时间或抓取时间
- 基础互动数据：点赞、评论、收藏、分享、转发、热榜位置等
- 内容概要

## 2. 优先采集的信息

有条件时，优先补充：

- 完整逐字稿
- 视频字幕
- 封面文字
- 画面中出现的关键文字
- 高赞评论
- 评论区共识
- 话题标签
- 关键画面描述

## 3. 可选采集的信息

必要时可补充：

- 账号主页定位
- 同账号相关内容
- 同一事件的多平台版本
- 视频截图
- 相关新闻或原始事件来源

## 4. 没有逐字稿时怎么办

没有逐字稿，不代表不能进入候选。

如果具备以下信息，也可以进入日报候选：

- 标题清楚
- 封面信息明确
- 评论区有共识
- 互动数据明显
- 人工能概括出事件/现象/故事梗概
- 能判断其与 MAX 主线的关系

但如果连“它具体讲了什么”都无法判断，不能进入日报。

## 5. 最低可用标准

一条信息进入《MAX外部情报日报》前，至少要能回答四个问题：

1. 它讲的是什么事件、动作、现象或规律？
2. 为什么有人关心？
3. 外界有什么反馈、争议或结果？
4. 它能不能被 MAX 重新解释？

四个问题答不出来，暂不进日报。

# 六、情报类型

AI 识别全网信息时，先判断它属于哪一类。

## A 类：具体事件

有明确的人、事、场景和结果。

例如：

- 某明星恋情引发争议
- 某富豪婚礼细节被讨论
- 某品牌 VIP 活动翻车
- 某名流活动座位安排被分析
- 某世界杯包厢或贵宾区差异被讨论

判断重点：

> 这个事件背后是否能讲身份、价值感、审美、阶层或关系结果。

## B 类：具体行为动作

一个人在具体场景中做了某个动作，并引发他人判断。

例如：

- 约会时点菜先看价格
- 香水喷太重
- 拍酒店照只拍大堂
- 红酒不会拿杯
- 车钥匙故意放桌上
- 和服务员说话怯场

判断重点：

> 这个动作暴露了什么生活习惯、场景熟悉度或身份破绽。

## C 类：具体社会现象

不是单一事件，而是一类反复出现的现象。

例如：

- 老钱风泛滥
- 男生精致土
- 富人从奢侈品转向疗养、长寿和体验
- 奢华从消费转向权限
- 只会供养的男人吸引力下降
- 高端展示面被普通团队拍成流水线

判断重点：

> 这个现象是否能反映时代变化、用户痛点或价值感转化问题。

## D 类：社会规律

从外部事件中提炼出的稳定规律。

例如：

- 物质供给的边际吸引力下降
- 身份表达从显性炫耀转向隐性识别
- 高端消费从“买东西”转向“进入别人进不去的地方”
- 真正的高级越来越依赖场景熟悉度，而不是单品价格

判断重点：

> 这个规律能否帮助 MAX 讲出更高一层的判断。

## E 类：热点/明星/赛事/影视片段

外部公共事件或文化素材。

例如：

- 世界杯包厢区别
- 明星恋情争议
- 电影片段里的阶层压迫
- 时装周、电影节、F1 的社交规则
- 富豪家族公开事件

判断重点：

> 不能停在八卦或剧情，必须能转成 MAX 的现实判断。

## F 类：评论区共识

评论区出现大量相似评价，形成真实用户语言。

例如：

- “像第一次来”
- “太用力了”
- “一看就不常来”
- “不是高级，是硬装”
- “有钱但没品”
- “人不错，但没感觉”

判断重点：

> 评论区是否暴露了用户真实判断语言。

## G 类：观点说教

已经被别人讲成结论，但缺少具体事件、动作、反馈和现实抓手。

例如：

- 吸引力不是聊天聊出来的
- 展示面很重要
- 男人要有框架
- 高价值男人要有生活方式

判断重点：

> 默认不优先进入日报，除非能还原成具体事件、动作、现象或规律。

## H 类：知识科普

偏解释知识，不天然构成情报。

例如：

- 老钱风怎么穿
- 香水怎么选
- 红酒怎么拿
- 奢侈品品牌历史
- 酒店礼仪科普

判断重点：

> 只有能转成身份判断瞬间，才有进入日报的价值。

# 七、情报优先级

优先级从高到低：

```text
S级：具体事件 / 具体行为动作 + 强反馈 + 能被 MAX 重解释

A 级：具体社会现象 / 社会规律 + 明确目标用户关系

B 级：热点 / 明星 / 赛事 / 影视片段 + 能转成 MAX 判断

C 级：评论区共识 / 用户真实语言

D 级：观点说教 / 知识科普
```

D 级不等于完全没用。

但它们主要用于参考，不优先进入日报。

如果一条观点说教素材能被还原成具体场景、具体动作、具体社会现象，它可以升级。

# 八、检索方法

## 1. 总原则：不搜概念，搜暴露点

不要优先搜：

- 展示面重要
- 吸引力底层逻辑
- 高价值男人
- 两性认知
- 价值感
- 如何提升吸引力

这些词容易搜到大量说教型内容。

优先搜：

- 具体场景
- 具体动作
- 具体物件
- 外界评价
- 结果反馈
- 评论区共识

## 2. 基础检索公式

```text
场景词 + 动作词 + 评价词
场景词 + 物件词 + 翻车词
人群词 + 吐槽词 + 结果词
热点词 + 阶层词 + 细节词
人物/家族/品牌 + 场景词 + 争议词
影视剧名 + 阶层/身份/社交/名利场
```

## 3. 示例检索方向

```text
男生朋友圈 太装
男生酒店照 像第一次来
女生吐槽 男生约会
高端餐厅 男生 尴尬
男生香水 味太重
男生点菜 看价格
红酒 不会拿杯
车钥匙 放桌上 炫富
老钱风 翻车
富豪婚礼 细节
世界杯 包厢 富人
品牌VIP 晚宴 富豪
明星恋情 身份差距
电影节 座位 咖位
F1 paddock club 富人
继承之战 老钱 穿搭
```

## 4. 基础关键词方向

规则文档只保留关键词方向。

具体关键词、账号、平台来源，后续放进《关键词与来源池》。

基础方向包括：

- 约会细节类
- 展示面破绽类
- 情感反馈类
- 供养者模式类
- 审美误判类
- 老钱/新贵/富人阶层类
- 名利场/高端场景类
- 明星/绯闻/热点类
- 体育赛事/贵宾区类
- 影视片段/阶层关系类
- 直播连麦/情感案例类
- 原生家庭/人格模式类
- 时尚/设计/旅行/拍摄类
- B站/知乎/公众号深内容类
- 评论区共识类

# 九、信息质量判断

全网信息质量不一样。

进入日报前，要先判断它的可信度和使用方式。

## 1. 一手事件

来自当事人、现场视频、公开报道、平台原帖。

优先级最高。

可作为事件情报。

## 2. 平台热议

某件事在平台上被大量讨论。

可作为社会现象或舆论情报。

注意区分：

> 大量讨论不等于事实完全准确，但可以说明这个判断正在传播。

## 3. 评论区共识

评论区大量出现相似评价。

很有价值。

因为它提供的是用户真实判断语言。

## 4. 热度信号

热度不是唯一标准，但必须记录。

可参考的数据包括：

- 播放量
- 点赞
- 评论
- 收藏
- 分享
- 转发
- 热榜位置
- 是否多平台发酵
- 是否出现连续讨论

判断原则：

> 热度是加权项，不是准入项。

一条信息有热度，说明它有传播势能。

但一条信息能不能进入日报，首先看它有没有现实抓手，能不能被 MAX 重解释。

## 5. 同行观点

可参考，但不优先。

同行观点必须被还原成现实素材，不能直接搬运结论。

## 6. 无法验证的故事

例如匿名投稿、情感号案例、无法确认来源的经历。

可以作为典型场景参考。

不能当成真实事件写。

表达时应按“这类场景”处理，而不是按“确切事实”处理。

好故事可以用。

但使用边界必须清楚：

> 无法验证但故事很好，可以作为典型场景；不能作为事实事件。

## 7. 纯搬运内容

如果只是搬运别人的观点，没有新事件、无评论共识、无真实场景，不优先。

# 十、去重与合并规则

日报不是素材堆。

如果多个来源讲的是同一件事、同一个现象或同一个动作，应合并。

## 1. 同一事件多平台出现

合并成一条事件情报。

来源处注明多个平台。

## 2. 同一现象反复出现

合并成一个社会现象。

例如多条内容都在讲：

> 男生高端餐厅拍照显得不常来。

则合并为：

> 高端餐厅展示中的“不常来”现象。

## 3. 同一动作反复被吐槽

合并成一个价值感破绽。

例如：

- 点菜先看价格
- 香水味太重
- 车钥匙放桌上

如果多平台多次出现，说明它不是单个吐槽，而是稳定识别点。

## 4. 同一观点不同账号重复讲

不重复进日报。

除非其中某条提供了新的事件、动作、评论区共识或更好的现实场景。

# 十一、KEEP / BUFFER / DROP

## KEEP

满足以下条件，优先进入日报：

- 有具体事件、动作、现象、规律或评论区共识
- 能回到 MAX 当前主线
- 目标用户会关心
- 能产生至少 2 个 MAX 可讲视角
- 不是纯说教、纯科普、纯热点搬运

KEEP 代表：

> 这条信息值得进入《MAX外部情报日报》，供 CHEN 判断是否继续进入选题系统。

## BUFFER

有价值，但暂不优先：

- 太像穿搭科普
- 太像情感号说教
- 太像同行话术
- 需要强转译才能回到 MAX
- 有现象，但场景还不够具体
- 只有局部观点，没有足够现实抓手

BUFFER 代表：

> 这条信息可以暂存，但不能直接变成 MAX 可口喷材料。

## DROP

直接丢弃：

- 与 MAX 主线无关
- 只有抽象观点，没有现实抓手
- 低俗、油腻、阴沟化
- 只有情绪，没有判断
- 纯知识科普，无法转成现实场景
- 纯热点八卦，无法重解释
- 事实过于模糊，无法判断讲的是什么

# 十二、《MAX外部情报日报》模板

日报每条使用以下格式。

```text
01｜标题

事件/现象/故事梗概：
把“发生了什么 + 反映了什么现象 + 大概故事”合在一起写。
不要拆太细。

热度与来源：
平台 / 链接 / 来源账号 / 检索关键词 / 内容发布时间 / 抓取时间 / 播放 / 点赞 / 评论 / 收藏 / 分享 / 转发 / 热榜位置 / 是否在发酵。
没有数据就写“未提供数据”。

评论区核心声音：
有就写，没有就空着。

MAX可讲视角：
列 2-5 个切口。
```

日报只负责让 CHEN 快速判断：

> 这条外部信息有没有可能被 MAX 讲。

日报不负责：

- 生成标题
- 生成最终选题
- 写口播
- 直接入 MAX 选题库

# 十三、日报与选题库边界

## 1. 日报是什么

日报是：

> 外部世界发生了什么。

日报可以多一点。

日报允许 KEEP 和 BUFFER 共存。

日报主要服务 CHEN 人工判断。

## 2. 选题库是什么

选题库是：

> MAX 可以讲什么。

选题库必须少而准。

只有经过二次筛选、能让 MAX 直接口喷的卡片，才能进入选题库。

入选题库前，必须经过《选题规则》二次处理。

## 3. 边界总则

```text
全网情报系统：发现现实素材。
MAX外部情报日报：整理外部世界发生了什么。
选题规则：把外部情报压缩成 MAX 可口喷卡片。
MAX选题库：存放最终可口喷卡片。
10-MAX内容生产 Skill：承接口喷并成稿。
```

# 十四、本文档不写什么

为了保持规则清晰，以下内容不写进本文档。

## 1. 不写完整关键词池

具体关键词会高频变化。

后续进入《关键词与来源池》。

## 2. 不写具体账号池

账号会不断新增、删除、调整优先级。

后续进入《关键词与来源池》。

## 3. 不写平台采集技术

例如：

- 抖音如何抓取
- 小红书如何抓取
- 视频号如何抓取
- 微博如何抓取
- OCR / ASR / 截图 / 评论采集如何执行

这些后续进入《平台采集执行方案》。

## 4. 不写 Agent 字段协议

Agent 字段协议等判断稳定后再写。

当前阶段先把人的判断练准。

## 5. 不写每日情报结果

每日结果进入《MAX外部情报日报》。

# 十五、当前版本总则

本规则当前只解决一个问题：

> 如何从全网信息里，找到能让 MAX 讲出自己判断的现实素材。

判断顺序是：

```text
先看有没有现实抓手。
再看是否能回到 MAX 主线。
再看目标用户是否关心。
最后看是否值得进入日报。
```

最重要的一句话：

> 先把人的判断练准，后续再让 Agent 继承人的判断。
