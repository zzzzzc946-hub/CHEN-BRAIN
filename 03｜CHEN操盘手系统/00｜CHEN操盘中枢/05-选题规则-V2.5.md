# 05-选题规则 V2.5

版本：V2.5｜外部情报压缩漏斗版 + 同质素材去重与合并机制 + 论证弹药嵌入机制  
定位：把外部情报日报压缩成 MAX 可读、可判断、可直接口喷的材料卡片

---

## 一、文档定位

本文件负责一件事：

```text
把《MAX外部情报日报》里的 KEEP / BUFFER 情报，
压缩、校准成 MAX 可读、可判断、可口喷的材料卡片。
```

它位于：

```text
04-MAX外部情报日报
↓
05-选题规则
↓
06-MAX选题库调试记录
↓
07-MAX选题库
↓
MAX口喷
↓
10-MAX内容生产 Skill
```

一句话：

```text
05 不是存情报，也不是写最终内容；
05 是把情报变成 MAX 能直接讲的材料。
```

V2.5新增：

```text
当一条材料存在真正匹配的外部论证时，
05 必须按《17-MAX观点论证弹药系统》完成预匹配，
并把已经转译成MAX人话的论证，直接嵌入MAX可口喷卡片。
```

论证后台可以单独检索和核验。

但最终前台呈现不能独立成报告。

## 二、上位校准标准

选题规则必须同时服从两层标准。

### 1. 长期最高标准

MAX 一直研究的是：

```text
真实价值如何形成价值感，并最终转化为吸引力和关系结果。
```

因此，一条材料是否值得进入选题漏斗，核心看它能不能帮助 MAX 解释：

```text
真实价值
价值表达
价值感形成
身份识别
审美判断
生活方式
展示面
吸引力
关系反馈
关系结果
目标用户自检
```

这些不是封闭清单，而是判断维度。

### 2. 当前阶段入口

当前阶段，MAX 用：

```text
高端展示面
```

作为可见入口，服务：

```text
有真实价值，但没有被正确看见的高价值男性。
```

所以，优先进入选题漏斗的材料，应该能帮助这个用户意识到：

```text
我不是没价值；
而是我的价值没有被正确表达、正确感知、正确转化。
```


## 四、选题漏斗

日报情报进入 07 前，必须经过以下六步。

### 1. 材料压缩

先把日报条目压缩清楚：

```text
这条材料到底讲了什么？
它是事件、现象、故事，还是一类反馈？
它被讨论的原因是什么？
外界的反应是什么？
```

不要急着写标题。

不要急着做结论。

先把材料本身压清楚。

---

### 2. 用户相关性判断

判断它和 MAX 当前目标用户有什么关系：

```text
有真实价值但没有被正确看见的男人，会不会关心？
他会不会自检？
他会不会觉得自己也犯过？
这是否关系到展示面、身份表达、审美表达、社交表达？
这是否能帮他理解自己为什么没有被正确感知？
```

目标用户不关心，不进入 07。

---

### 3. MAX 可讲性判断

判断 MAX 能不能借这个材料讲出自己的判断。

MAX 不能只是复述资讯。

必须能讲出：

```text
为什么这个现象成立
为什么这个表达会失败
为什么这个细节会改变外界判断
为什么真实价值没有被正确感知
为什么有些人越展示越掉价
为什么有些人不刻意展示反而更有价值感
```

如果只能讲成热点评论、情感鸡汤、穿搭科普、炫富八卦、追女生技巧，而不能回到价值感主线，不进入 07。

---

### 4. 切口能量校准

判断这个材料的讲法是否符合 MAX 的表达气质。

外部情报可以来自：

```text
翻车
破绽
尴尬
误判
低位
供养
审美断层
场景不熟
```

但 MAX 的最终切口不能停留在：

```text
羞辱
批判
低位审判
穷人视角
传统情感号式压迫感
```

切口能量优先级：

```text
正向价值提供
↓
知识 / 品味 / 生活方式启发
↓
具体场景建议
↓
问题诊断
↓
刺痛 / 羞辱 / 批判
```

核心规则：

```text
抓取可以抓破绽。
日报可以记录破绽。
选题库卡片必须尽量转成正向价值、知识启发、品味判断和场景建议。
```

---

### 5. 论证弹药预匹配

只有在论证能显著增强MAX判断时，才进入：

```text
17-MAX观点论证弹药系统
```

先回答：

```text
这张卡片真正需要被支持的是哪一句判断？
需要的是实证、机制、历史案例，还是文化类比？
MAX自有案例是否已经足够？
```

可检索范围：

```text
1. 社会学 / 人类学
2. 心理学 / 行为科学
3. 历史 / 名人 / 商业人物
4. 影视 / 文学
5. 艺术 / 美学 / 建筑 / 音乐
6. MAX自有案例 / 用户反馈
```

硬规则：

```text
六类是检索范围，不是固定填空。
一张卡片只保留最强的1—2个论证。
没有真正贴合的论证，可以不加。
论证资源不作为选题成立的门槛。
不能因为某个选题能找到很多名人和实验，就判断它是好选题。
```

日报卡片阶段生成的是预论证。

MAX口喷后，10号 Skill 必须根据最终观点进行二次校准。

---

### 6. 入库判断

最后判断：

```text
这条材料是否能让 MAX 看完就想讲？
```

如果还需要解释半天背景，暂不入库。

如果 MAX 看完不知道讲什么，暂不入库。

如果论证比MAX观点更抢眼，删除论证或进入06调试。

如果 CHEN / MAX 明显感觉方向不对，进入 06 调试。

## 五、MAX 可口喷卡片格式

进入 `07-MAX选题库` 的内容，统一使用以下格式：

```text
【MAX可口喷卡片】

来源信息：
平台 / 链接 / 热度 / 来源账号

话题 / 母题：
这条材料归属的大方向。

材料摘要：
把具体事件 / 现象 / 故事 / 结果 / 反馈 / 争议合并写清楚。

目标用户为什么会关心：
这和 MAX 当前阶段用户有什么关系。

MAX 可讲切入点：
1.
2.
3.
4.
5.

推荐切入点：
当前最值得优先给 MAX 口喷的方向。

【可穿插论证】

论证一｜建议放在开头 / 中段 / 结尾 / 后台
它支撑的判断：
明确写出它支撑卡片里的哪一句MAX判断。

MAX可以直接这样讲：
把理论、史实、实验、人物、作品或艺术案例，转译成MAX能直接说的人话。

论证二｜建议放在开头 / 中段 / 结尾 / 后台
它支撑的判断：
...

MAX可以直接这样讲：
...

【论证使用边界】
说明哪些只是类比，不能说成“证明”；
哪些出处需要后台保留；
哪些只适合帮助MAX理解，不建议直接口喷。
```

填写要求：

```text
材料摘要必须写具体，不写空泛观点。
切入点不是标题，不写成标题党。
推荐切入点只能选一个。
论证只能保留最贴合的1—2个。
前台必须是MAX能直接讲的话，不只是学者、理论和作品名称。
后台保留来源类型、证据等级、原始出处、核验状态和争议说明。
卡片只服务 MAX 口喷，不混入发布管理字段。
```

如果没有真正可用的论证，写：

```text
【可穿插论证】
本条不强行添加。真实材料和MAX判断已经足够成立。
```

禁止为了格式完整硬凑。

## 七、入库与淘汰规则

### 1. 可以进入 07 的材料

必须满足：

```text
有具体材料
有外部反馈或现实结果
目标用户会关心
能承载 MAX 判断
切口能量不低级
MAX 看完能直接口喷
论证如存在，已经完成基本核验和口语转译
```

### 2. 不进入 07 的材料

以下内容不入库：

```text
只有热度，没有场景
只有新闻事实，没有用户体感
只有抽象观点，没有具体动作或反馈
只能做资讯复述
和目标用户无关
情绪很大，但和价值感主线无关
能讲成泛情感、泛成功学、泛吐槽，但讲不成 MAX
AI 为了凑字段硬编出来的材料
靠伪名言、伪实验、伪史实制造专业感
只有论证知识，没有MAX自己的判断
```

### 3. 淘汰但有价值的材料

如果材料不能入库，但调试过程有价值，记录到 `06-MAX选题库调试记录`。

记录目的不是保留垃圾材料，而是沉淀：

```text
为什么它不能入库
哪里判断错了
以后如何避免
是否需要反向更新 01 / 02 / 03 / 05 / 10 / 13 / 16 / 17
```

## 十、最小输出原则

选题规则不是为了让文档看起来完整，而是为了让 MAX 更容易产出内容。

每次整理外部情报时，优先输出：

```text
少而准
可口喷
有具体材料
有反馈
有判断空间
论证能直接使用
```

不要为了数量堆卡片。

不要为了格式完整硬凑字段。

不要加入复杂评分系统。

最重要的是：

```text
每一张进入选题库的卡片，都必须提升 MAX 口喷内容的准确度。
```

## 十一｜04 到 07 之间必须增加三层去重

### 1. 原始重复

出现以下任一情况，视为同一素材：

```text
同一作品不同分享链接
同一标题 + 同一逐字稿
同一发布时间 + 同一内容结构
重复提交但互动数据更新
```

处理方式：

```text
04 不重复新增。
保留最新链接和最新互动数据。
历史链接和历史数据作为更新记录保留。
07 不重复生成卡片。
```

### 2. 语义重复

不同作品如果在讲同一个现象、同一种用户问题和同一个底层判断，视为同质素材。

例如：

```text
普通场景比高端场景更自然
咖啡馆适合拍生活化展示面
展示面要有活人感
一次性高消费容易显得装
```

这些可以在 04 中作为不同来源保留，但进入 07 前必须合并判断，不能一条来源生成一张卡片。

### 3. 视角重复

即使来源和表面话题不同，只要推荐切入点最终仍然是同一句话，也视为重复。

判断问题：

```text
这条素材是否提供了新的用户痛点？
是否提供了新的具体场景？
是否提供了新的因果机制？
是否提供了新的可执行判断？
是否能让 MAX 讲出一条此前不能讲的内容？
```

五项全部为否时，不新增 07。

---

## 十二｜04 存来源，07 存判断

```text
04-MAX外部情报日报：按外部作品和现象保存。
07-MAX选题库：按 MAX 最终可讲判断保存。
```

因此允许：

```text
多条 04 素材
↓
合并成一张 07 卡片
```

禁止：

```text
每一条 KEEP
↓
机械生成一张 07 卡片
```

KEEP 只代表素材值得进入日报，不代表必须进入选题库。

---

## 十三｜多来源卡片写法

当多个外部素材共同支撑同一判断时，07 的“来源信息”可以写多个来源。

要求：

```text
保留主来源。
补充高热度或高代表性的辅助来源。
不需要把所有重复来源全部塞进卡片。
材料摘要写共同现象，不逐条复述每个视频。
推荐切入点只能保留一个。
论证只围绕合并后的独立判断，不按来源数量增加。
```

---

## 十四｜BUFFER 的新使用方式

以下内容优先进入 BUFFER：

```text
热度很高，但没有新判断。
与已有卡片高度重合。
只能作为趋势补证。
能补充例子，但不能独立支撑一条内容。
同类排行榜、模板清单、泛技巧内容。
论证素材不错，但还没有找到准确MAX切口。
```

BUFFER 后续有四种去向：

```text
补到已有 07 卡片的来源信息。
补到已有卡片的材料摘要。
补到已有卡片的论证后台来源。
形成新的具体机制后，再独立入库。
```

---

## 十五｜07 卡片限流规则

每轮批量处理结束后，必须执行一次横向比较。

```text
先看所有候选卡片。
再按用户痛点和底层判断聚类。
每个核心判断只保留一张最强卡片。
其余来源降为辅助证据或 BUFFER。
```

优先保留：

```text
用户一看就能自检
具体场景明确
MAX 看完能直接讲
有独立机制
与现有卡片差异明显
能服务当前“高端展示面”入口
论证能自然嵌入而不抢主体
```

不因为热度高就单独入库。

不因为能找到名人或实验就单独入库。

---

## 十六｜论证弹药最终总则

```text
论证规则可以单独写在17号。
论证检索和核验可以在后台独立完成。
但最终呈现必须穿插在MAX可口喷卡片里面。
```

最重要的判断：

```text
MAX不是靠引用别人显得有涵养。
MAX的涵养，是能把历史、人性、社会机制、审美和用户真实场景打通。
```

## 十七｜本轮总则

```text
来源数量可以增加。
日报可以变厚。
论证候选可以很多。
选题库不能随来源和论证数量同比膨胀。
```

最重要的两句话：

```text
04 的单位是外部素材。
07 的单位是 MAX 能讲出的独立判断。
```
