# 01｜选题规则.md｜V2.0

## 0｜文档定位

本文档是 MAX 内容生产中关于「话题、选题、标题、判断、入库」的最高执行规则。

它负责回答：

> 如何把用户画像里最能打动用户的点，凝聚成一句话、一个观点、一条可以拍的视频。

用户画像回答：

> 对谁说，什么最能打动他。

选题规则回答：

> 怎么把这个点变成话题、选题、标题和判断。

选题库回答：

> 哪些选题已经足够准确，可以进入内容生产。

当前选题部保持双文档结构：

```text
01｜选题规则.md
02｜MAX选题库.md
```

其中：

```text
01｜选题规则.md = 生产方法
02｜MAX选题库.md = 最终结果
```

用户画像作为操盘手系统独立文档存在，不放入选题部。

---

## 1｜选题生产最高原则

MAX 的选题不能先从 MAX 理论出发。

不能先问：

```text
这条符不符合 MAX 价值观？
这条能不能表达价值感？
这条是不是高级？
这条能不能体现 MAX 的思想？
```

必须先问：

```text
这条内容是给哪类用户看的？
他现在最痛的是什么？
这个痛点属于最痛、次痛还是泛痛？
他为什么会停下来听？
他为什么会觉得这说的是自己？
他为什么会因此更想找 MAX？
```

正确顺序是：

```text
用户画像
↓
用户痛点
↓
具体场景
↓
MAX 判断
↓
选题
↓
标题
↓
判断骨架
↓
入库 / 废弃
```

错误顺序是：

```text
MAX 理论
↓
抽象观点
↓
强行找场景
↓
包装标题
```

冷启动阶段，所有选题必须先过用户画像，再过 MAX 价值观。

顺序不能反。

因为内容不是为了证明 MAX 理论正确，而是为了让目标用户感到：

```text
他说的是我。
他懂我卡在哪里。
他知道为什么我之前没效果。
他能判断我该怎么做。
```

## 2｜当前内容母题

当前阶段内容母题是：

> 怎么打造高级展示面。

内部可以理解为：

```text
如何高级地展示自己的价值
如何低调地装逼
如何展示条件但不显得装
如何让别人自然高看你
```

但对外不能粗暴说内部理解。

对外要转译成用户能接受的表达：

```text
怎么发才不显得装
怎么拍才不像流水线
怎么让朋友圈更高级
为什么普通展示面吸引不了高质量女生
为什么你有价值但女生没有感受到
为什么你花钱以后还是像在讨好
```

当前内容母题服务于 MAX 更大的 IP 母题：

> 价值如何形成价值感，并最终影响关系结果。

但冷启动阶段不能直接讲大母题。

冷启动阶段必须把大母题压到具体业务里：

```text
展示面怎么做才高级
展示面怎么发才高级
展示面为什么没效果
普通展示面为什么解决不了高价值用户的问题
MAX 为什么能判断这些问题
```

## 3｜当前阶段内容比例

当前阶段，内容必须以展示面为主。

建议比例：

```text
70% 以上围绕展示面
30% 以内穿插审美、生活方式、情感、社交判断
```

但穿插内容也必须尽量回到展示面。

例如：

```text
情感问题 → 回到女生看完朋友圈后怎么判断你
审美问题 → 回到照片、配文、场景、朋友圈呈现
生活方式问题 → 回到你如何展示生活而不显得装
社交问题 → 回到你的线上第一判断是否建立
```

不能因为“不一直聊照片”，就扩成：

```text
泛情感号
泛社交号
泛生活方式号
聊天技巧号
男性觉醒号
```

## 4｜用户画像调用规则

每次生成选题前，必须先调用《用户画像.md》。

不是形式上调用，而是必须明确三件事：

```text
1. 目标用户是哪一类？
2. 打的是哪一层痛点？
3. 具体痛点 / 需求是什么？
```

用户分为三类：

```text
一类用户：已经相信展示面有效，并且拿到过任何正反馈的人。
二类用户：没有真正找专业团队拍过，但已经知道照片和展示面有用的人。
三类用户：完全没有展示面概念，但有情感和社交痛点的人。
```

注意：

```text
只要拍过展示面，或者做过类似展示升级，并且拿到过任何正反馈，都归入一类用户。
```

不要再纠结：

```text
明显结果
不明显结果
小结果
大结果
```

只要有正反馈，就算结果。

## 5｜痛点层级调用规则

用户痛点分为三层：

```text
最痛痛点
次痛痛点
泛痛点
```

当前阶段优先打最痛痛点。

次痛痛点可以用于扩展。

泛痛点可以作为场景补充，但不能成为内容主线。

判断一条选题是否优先，要先判断它打的是哪一层痛点。

例如：

```text
一类用户最痛痛点：
普通展示面拍不出他的段位，吸引不到他想要的高质量女生。

一类用户次痛痛点：
普通团队太流水线、太多人拼、太像批量拍摄，缺少私密感和定制感。

一类用户泛痛点：
不知道朋友圈几点发，不知道配文怎么写，不知道哪张图该发。
```

优先级：

```text
最痛痛点 > 次痛痛点 > 泛痛点
```

如果一条内容只打泛痛点，除非场景非常强，否则不作为冷启动优先选题。

## 6｜选题前置三问

每条选题生成前，必须先回答：

### 1. 这条内容给谁看？

必须明确：

```text
一类用户 / 二类用户 / 三类用户
```

不能只说“目标男性”“高价值男性”“优质客户”。

### 2. 这条内容打哪里？

必须明确：

```text
最痛痛点 / 次痛痛点 / 泛痛点
```

不能只说“用户会关心”。

### 3. 这条内容具体解决什么痛点或需求？

必须写清楚具体痛点。

例如：

```text
普通展示面拍不出他的段位
普通展示面吸引不到高质量女生
朋友拍不出专业展示面的效果
他有价值但女生没有高看他
他不想成为供养者
他想展示条件但怕显得装
```

如果这三问答不清楚，这条选题不成熟。

## 7｜什么是话题

话题是：

> 一个可以继续展开出多条内容的方向。

话题不是一条视频本身。

话题可以大，也可以小，但它必须能继续长出多条选题。

例如：

```text
高级展示面怎么发
展示面为什么没效果
普通展示面为什么 low
朋友圈怎么发才高级
五星酒店怎么发才不装
下午茶怎么发才高级
旧朋友圈怎么处理
拍完展示面以后怎么用
为什么朋友拍不出专业展示面
为什么你总是变成供养者
```

判断一个东西是不是话题，不看它长短，而看它还能不能继续长出多条内容。

如果它还能继续展开，它就是话题。

如果它已经明确这一条视频具体讲什么，它就是选题。

---

## 8｜什么是选题

选题是：

> 这一条视频到底在讲什么。

选题必须是一句话。

选题不是解释段落。

选题不是一整段逻辑。

选题也不是标题堆砌。

选题可以是一个问题：

```text
五星酒店下午茶怎么发，才不会让别人觉得你在装？
```

选题可以是一个观点：

```text
很多人不是没有好照片，是不会把好照片发得高级。
```

选题可以是一个方法：

```text
拍完展示面以后，第一步不是发朋友圈，而是先处理旧内容。
```

选题可以接近标题。

但选题和标题功能不同。

选题负责讲清楚内容方向。

标题负责让用户停下来。

---

## 9｜选题不能写成解释

错误选题示例：

```text
很多人以为展示面拍好了就结束了，但真正让朋友圈看起来高级的，不只是照片本身，而是你怎么选图、怎么配文、怎么定位、怎么排版、怎么处理旧内容。照片拍得再好，发错了也会让别人觉得你在装。
```

这不是选题。

这是解释。

正确选题应该压缩成一句话：

```text
拍完展示面以后，怎么发才不会显得在装？
```

或者：

```text
很多人不是没有好照片，是不会把好照片发得高级。
```

---

## 10｜选题必须从用户痛点长出来

不要从大词长选题。

不要从这些词直接生成选题：

```text
价值感
高级
审美
展示面
生活方式
社交资产
吸引力
情感结果
```

这些词可以作为内部理解，但不能直接成为选题入口。

选题要从用户真实问题长出来：

```text
我这样发会不会显得装？
我拍了为什么没用？
我有照片但不知道怎么发。
我配文会不会太土？
我旧朋友圈是不是拖后腿？
我想展示条件但不想炫耀。
我为什么加了微信以后女生反应很冷？
我为什么请她吃饭、买单、送礼，她还是不高看我？
我为什么找朋友拍了照片，但还是没什么效果？
我为什么花钱拍展示面，还是拍不出我想要的段位？
```

用户痛点越具体，选题越容易成立。

## 11｜当前高优先级痛点方向

以下痛点方向在冷启动阶段优先：

### 1. 一类用户｜普通展示面拍不出他的段位

核心问题：

```text
为什么普通展示面拍不出高净值用户要的感觉？
为什么普通展示面看起来很流水线？
为什么多人排队拍、多人 AA 场景，会让一类用户天然排斥？
为什么他不想要批量拍摄，而想要更私密、更定制的展示面？
```

### 2. 一类用户｜普通展示面吸引不到高质量女生

核心问题：

```text
为什么普通照片只能带来普通反馈？
为什么普通展示面吸引来的女生质量不够？
为什么高分女生不是看你有没有照片，而是看你像不像那个层级的人？
```

### 3. 一类用户｜有价值但没有被正确看见

核心问题：

```text
为什么你有钱但不显贵？
为什么你有经历但没被看见？
为什么你有资源但展示出来很普通？
为什么女生看完以后还是没有高看你？
```

### 4. 二类用户｜知道照片有用，但不知道为什么要找专业团队

核心问题：

```text
朋友也能拍照，为什么要找专业展示面团队？
为什么现在随便拍几张帅照已经不够？
为什么普通帅照和高级展示面不是一回事？
为什么展示面不是拍照，而是线上竞争力？
```

### 5. 二类用户｜社交环境越来越卷，普通照片不够用了

核心问题：

```text
为什么以前随便拍几张还能用，现在不行了？
为什么女生眼光越来越高以后，普通照片没竞争力？
为什么你以为自己拍得还行，其实在高质量女生眼里很普通？
```

### 6. 三类用户｜供养者切口

核心问题：

```text
为什么你对女生越好，她越不把你当回事？
为什么你愿意花钱，但女生没有高看你？
为什么你付出很多，最后却像在补分？
为什么不是你不会聊天，而是她一开始就没把你放高？
```

### 7. 展示方式痛点｜想展示但怕显得装

核心问题：

```text
五星酒店怎么发才不像装？
下午茶怎么发才不像拼多多名媛？
旅行照怎么发才不像打卡？
豪车怎么发才不像蹭车？
高级餐厅怎么发才不像第一次来？
朋友圈怎么发才不像在证明自己过得很好？
```

## 12｜什么是标题

标题是：

> 把选题里最能让用户停下来的点拎出来。

标题不是选题的完整复述。

标题也不是把内部价值观直接说出来。

标题要让用户产生：

```text
这说的不就是我吗？
我是不是也这样？
为什么会这样？
这个问题我以前没意识到。
我想知道他后面怎么讲。
```

标题必须说用户真正关心的问题。

不要制造伪痛点。

例如：

```text
女生一眼就知道你不常来。
```

这句话有刺，但未必是用户真正关心的问题。

用户真正关心的可能是：

```text
我这样发，会不会让别人觉得我在装？
```

所以标题要贴真实担心，不要只追求一句话看起来狠。

---

## 13｜标题和选题的区别

选题负责回答：

```text
这条视频到底讲什么？
它解决客户什么问题？
MAX 的判断是什么？
为什么这个问题值得讲？
```

标题负责回答：

```text
用户为什么点进来？
用户为什么觉得这和自己有关？
用户为什么想继续听？
```

一句话区分：

```text
选题是内容方向。
标题是传播钩子。
```

例如：

话题：

```text
高级展示面怎么发
```

选题：

```text
五星酒店下午茶怎么发，才不会让别人觉得你在装？
```

标题：

```text
五星酒店下午茶，发错了真的很像拼多多名媛。
```

## 14｜什么是判断

判断不是审核意见。

判断是：

> 这条视频的用户归属、痛点归属、核心观点和文案骨架。

每条选题的判断必须写清楚六件事：

```text
1. 核心观点
2. 目标用户
3. 痛点层级
4. 具体痛点 / 需求
5. 判断骨架
6. 内容目的
```

如果这六件事写不清楚，选题不能进入选题库。

---

## 15｜判断标准格式

每条候选选题统一使用以下判断结构：

### 核心观点

这条视频真正要表达的判断是什么。

### 目标用户

这条内容主要给哪类用户看。

必须写：

```text
一类用户 / 二类用户 / 三类用户
```

### 痛点层级

这条内容打的是：

```text
最痛痛点 / 次痛痛点 / 泛痛点
```

### 具体痛点 / 需求

这条内容具体打中用户什么痛点或需求。

必须具体。

不能写成：

```text
用户想变高级。
用户想有结果。
用户想提升吸引力。
```

要写成：

```text
一类用户不满意普通展示面太流水线，拍不出自己的段位。
二类用户知道照片有用，但还没被说服为什么要找专业团队。
三类用户在关系里变成供养者，花钱和付出都没有被高看。
```

### 判断骨架

这条视频后面怎么讲。

至少列出 2-4 个展开点。

判断骨架必须能直接支撑脚本生成。

### 内容目的

用户看完以后，应该产生什么认知变化。

例如：

```text
让用户意识到：普通展示面不是不好看，而是不足以匹配他的段位。
让用户意识到：朋友拍照和专业展示面之间差的不是相机，而是判断。
让用户意识到：自己成为供养者，不是因为不该花钱，而是前置判断没有建立。
```

## 16｜问题型选题的判断写法

如果选题是问题型，例如：

```text
五星酒店下午茶怎么发，才不会让别人觉得你在装？
```

判断要像 MAX 当面回答客户。

示例：

### 核心观点

高级场景不是不能发，而是不能发得像第一次来。

### 目标用户

一类用户 / 二类用户。

### 痛点层级

次痛痛点。

### 具体痛点 / 需求

用户想展示条件，但怕显得装、显得第一次来、显得像打卡。

### 判断骨架

```text
第一，不要把所有东西都发出来，越想证明自己来了，越像第一次来。
第二，配文要克制，不要写“生活需要仪式感”这种用力句。
第三，照片重点不要全放在场景上，要让别人觉得这是你的生活，而不是你为了拍照临时进入的地方。
第四，高级不是不展示，而是展示得像日常。
```

### 内容目的

让用户意识到：

> 高级展示面不是拍到高级场景就结束，而是要知道怎么发，才不会让自己的价值掉下来。

---

## 17｜观点型选题的判断写法

如果选题是观点型，例如：

```text
很多人不是没有好照片，是不会把好照片发得高级。
```

判断要负责论证观点。

示例：

### 核心观点

照片好只是素材，高级感来自你怎么使用素材。

### 目标用户

一类用户 / 二类用户。

### 痛点层级

次痛痛点或泛痛点。

### 具体痛点 / 需求

用户有照片，但不知道怎么选图、配文、排版、发布，导致好照片没有发挥价值。

### 判断骨架

```text
第一，同一组照片，选图不同，别人对你的判断不同。
第二，配文太用力，会把本来高级的照片拉低。
第三，旧朋友圈不处理，新照片发上去也救不回来。
第四，MAX 判断的不是单张照片好不好看，而是整套东西发出去以后，别人会把你理解成什么人。
```

### 内容目的

让用户意识到：

> 照片不是结果，照片只是素材，真正重要的是怎么用。

---

## 18｜纠错型选题的判断写法

如果选题是纠错型，例如：

```text
很多人展示面没效果，不是照片少，而是无效动作太多。
```

判断要回答：

```text
错在哪里？
为什么错？
怎么改？
```

示例：

### 核心观点

不是所有展示动作都有价值，只有能改变别人判断的动作才有价值。

### 目标用户

一类用户 / 二类用户。

### 痛点层级

最痛痛点或次痛痛点。

### 具体痛点 / 需求

用户做了展示面，但没有获得想要的反馈，不知道问题出在拍摄、发布还是整体判断。

### 判断骨架

```text
第一，拍了很多照片，但没有统一的朋友圈感觉。
第二，每张照片都想证明自己过得不错。
第三，照片拍完以后乱发，素材没有被真正用起来。
第四，展示面不是照片仓库，而是别人判断你是什么人的入口。
```

### 内容目的

让用户意识到：

> 展示面不是做了就有效，而是要做对、发对、被正确理解。

---

## 19｜“你不知道，我告诉你”型选题

这类选题可以少量穿插。

它的作用是制造一种感觉：

> 原来这个细节还有判断。

适合内容：

```text
为什么不要把菜单拍得太清楚？
为什么定位不能乱带？
为什么高级餐厅不要九宫格全发？
为什么有些照片越修越廉价？
为什么有些下午茶照片一看就像打卡？
为什么朋友圈不是照片仓库？
为什么旧朋友圈要先处理？
```

这类选题不能只是冷知识。

必须最后回到：

```text
这个细节会让别人怎么判断你。
```

如果只是炫知识，不进入选题库。

## 20｜供养者相关选题规则

供养者可以作为三类用户的重要情感痛点切口。

但不能把 MAX 做成普通情感号。

供养者不是简单的“舔狗”。

供养者是：

> 有资源、有付出能力、愿意花钱、愿意请客送礼、愿意提供情绪价值，但在关系里没有被高看，反而被当成钱包或工具的人。

供养者选题不能停留在：

```text
不要给女生花钱
不要当舔狗
不要提供情绪价值
不要被女人拿捏
```

这些是普通情感号。

MAX 要讲的是：

> 为什么你有资源、有付出，但因为前置判断没建立，最后所有付出都像在补位置。

所以供养者切口必须回到：

```text
女生一开始有没有高看你
你的价值有没有在见面前被看见
你的朋友圈有没有建立前置判断
你是不是只能靠后续付出去补前面没建立起来的位置
展示面有没有让你在关系开始前就进入更高位置
```

示例方向：

```text
为什么你越请客，她越不把你当回事？
为什么你花钱以后，反而像在补分？
为什么不是你不会聊天，而是她从一开始就没高看你？
为什么你有价值，但只能靠付出去换关系？
```

## 21｜“拍过没用”相关规则

“拍过没用”不是独立用户类型。

它是痛点 / 异议 / 内容切口。

它可以归入：

```text
一类用户：拍过，有正反馈，但不满意，想升级。
二类用户：没专业拍过，但担心花钱以后也没效果。
三类用户：不懂展示面，但可以通过“为什么你朋友圈没效果”被唤醒。
```

所以后续不要再新增“拍过没用用户”这个用户类型。

但可以保留“拍过没用”作为高频选题方向。

例如：

```text
为什么你拍过展示面，还是没有吸引到想要的女生？
为什么你拍了很多照片，朋友圈还是没什么变化？
为什么普通展示面看似好看，但没有改变女生对你的判断？
```

## 22｜内部词与外部表达

以下词可以作为内部理解：

```text
虚荣心
低调装逼
高级地装逼
供养者
价值感
前置判断
价值被看见
```

但对外表达要转译。

### 内部：低调装逼

外部：

```text
展示条件，但不显得装。
```

### 内部：满足虚荣心

外部：

```text
让别人自然看见你的价值。
```

### 内部：前置判断

外部：

```text
女生见你之前，已经通过朋友圈决定怎么对待你。
```

### 内部：价值感没有形成

外部：

```text
别人看完以后，没有更想了解你。
```

内部词不能直接外露。

否则内容会变俗、变硬、变成普通情感号或男性觉醒号。

## 23｜标题禁区

以下标题方向慎用或禁用：

```text
靠性别对立制造爽感
替男人出气
攻击女性
只讲“男人凭什么”
只讲“女人太现实”
只讲“别当舔狗”
只讲“买单的人不需要情绪价值”
```

例如：

```text
买单的人不需要提供情绪价值。
```

这句话有暴论感，但不是 MAX 的作风。

它容易把账号带成：

```text
男性觉醒号
情感对立号
替男人出气的情绪号
```

MAX 更应该讲：

```text
为什么你明明愿意花钱，女生还是没有高看你？
为什么你花了钱，反而像在讨好？
为什么她看完你朋友圈以后，就没有把你放到更高的位置？
```

MAX 的锋利不是骂女人，也不是替男人出气。

MAX 的锋利是：

> 把男人为什么拿不到想要反馈的原因讲准。

## 24｜否决门

出现以下情况，默认不进入选题库：

```text
1. 不是从用户痛点出发。
2. 说不清楚对应哪一类用户。
3. 说不清楚打的是最痛、次痛还是泛痛点。
4. 选题写成解释段落。
5. 只有现象，没有原因和解决方向。
6. 只有 MAX 理论，没有具体用户场景。
7. 只有观点，没有判断骨架。
8. 像普通情感号。
9. 像普通摄影号。
10. 像聊天技巧号。
11. 像男性觉醒号。
12. 靠性别对立制造爽感。
13. 为了显得高级而牺牲用户理解。
14. 不能说明为什么这件事需要 MAX 判断。
15. 不能说明用户看完以后会获得什么具体判断。
```

## 25｜入库规则

只有达到以下标准的选题，才能进入《02｜MAX选题库.md》：

```text
1. 话题清楚。
2. 选题是一句话。
3. 标题有传播钩子。
4. 判断字段完整。
5. 目标用户明确。
6. 痛点层级明确。
7. 具体痛点 / 需求明确。
8. 判断骨架能支撑脚本生产。
9. 不偏离当前内容母题。
10. 不像普通情感号、摄影号、聊天号。
```

不成熟的选题不入库。

废弃选题不入库。

需要继续打磨的选题，可以暂时在对话中保留，不进入正式选题库。

## 26｜选题库标准格式

每条进入《02｜MAX选题库.md》的选题，统一使用以下格式：

```text
话题：
选题：
标题：
判断：
状态：
```

其中：

```text
话题 = 能继续长出多条内容的方向
选题 = 这一条视频到底讲什么
标题 = 让用户停下来的传播表达
判断 = 用户归属、痛点归属、核心观点、文案骨架
状态 = 过稿 / 打磨中 / 废弃
```

正式选题库原则上只保留：

```text
过稿
少量正在打磨中的高价值选题
```

废弃选题不进入选题库。

---

## 27｜判断字段标准格式

选题库中的「判断」字段必须按以下结构写：

```text
核心观点：
目标用户：
痛点层级：
具体痛点 / 需求：
判断骨架：
内容目的：
```

示例：

```text
核心观点：
高级场景不是不能发，而是不能发得像第一次来。

目标用户：
一类用户 / 二类用户。

痛点层级：
次痛痛点。

具体痛点 / 需求：
用户想展示条件，但怕显得装、显得第一次来、显得像打卡。

判断骨架：
1. 不要把所有东西都发出来，越想证明自己来了，越像第一次来。
2. 配文要克制，不要写“生活需要仪式感”这种用力句。
3. 照片重点不要全放在场景上，要让别人觉得这是你的生活，而不是你为了拍照临时进入的地方。
4. 高级不是不展示，而是展示得像日常。

内容目的：
让用户意识到：高级展示面不是拍到高级场景就结束，而是要知道怎么发，才不会让自己的价值掉下来。
```

## 28｜生产流程

以后生成选题时，按以下流程执行：

```text
1. 调用用户画像
2. 确认目标用户
3. 确认痛点层级
4. 确认具体痛点 / 需求
5. 找具体场景
6. 提炼 MAX 判断
7. 压成一句话选题
8. 生成标题
9. 写判断骨架
10. 审核是否入库
```

不要从标题开始。

不要从观点开始。

不要从 MAX 理论开始。

从用户痛点开始。

---

## 29｜一句话总则

MAX 冷启动阶段的选题，不是讲展示面理论。

而是：

> 从用户最具体的展示焦虑和情感痛点里，提炼出一个他真正关心的问题，再用 MAX 的判断把它讲准。
