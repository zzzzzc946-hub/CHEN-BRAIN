# CHEN 橙子内容采集助手设计

## 目标

把现有飞书多维表格采集器升级为一个可点击打开的本地桌面软件。软件不再默认依赖飞书作为前台界面，而是在软件页面内完成链接输入、平台登录、内容采集、逐字稿转写、结果保存、分类管理和导出。

飞书保留为后续可选导出目标，不作为第一版主流程。

## 第一版范围

第一版聚焦“本地可用、流程闭环、稳定复用现有采集能力”：

- 桌面入口：用户双击打开软件界面。
- 平台入口：首页展示抖音、小红书、B站、视频号平台卡片。
- 平台卡片交互：鼠标悬停发光，点击进入对应扒取页。
- 橙子视觉：顶部和首页使用可爱橙子形象；默认轻微浮动，鼠标悬停时变形榨汁循环；橙子会眨眼。
- 抖音第一优先：先实现抖音扒取页，再复用到小红书、B站、视频号。
- 链接模式：每个平台页支持“主页链接”和“单作品链接”切换。
- 登录流程：点击开始扒取后先尝试直接采集；如果登录态失效，再自动弹出对应平台登录页。
- 结果展示：采集结果直接显示在软件页面，不默认写入飞书。
- 本地记忆：结果保存到本地数据库，下次打开仍可查看。
- 表格管理：用户可以新建多个采集表格，用于分类管理结果。
- 多视图：同一张采集表支持表格视图、卡片视图、详情视图切换。
- 导出：支持导出 Word、Excel、CSV。

## 暂不做

- 不做云端账号系统。
- 不做多人协作。
- 不做素材文件长期仓库；封面、音频、视频仍以临时处理为主，最终保存结构化结果。
- 不默认同步飞书；飞书作为后续可选导出/同步能力。
- 不一次性实现复杂权限、模板市场、插件系统。

## 信息架构

### 首页

首页是 DyD 风格的平台选择页，强调“这是一个采集助手”：

- 顶栏：橙子图标、软件名、设置、首页、最小化、关闭。
- 主视觉：`CHEN 一个会榨取信息的橙子助手`。
- 主按钮：选择平台。
- 次按钮：历史记忆、导出文档。
- 平台卡片：
  - 抖音扒取：单作品链接、主页作品批量。
  - 小红书扒取：图文/视频笔记、主页笔记批量。
  - B站扒取：BV/av/分享链接、主页视频批量、长视频 ASR、导出文档。
  - 视频号扒取：单作品链接、封面和视频信息、登录态检测、失败原因提示。

### 平台扒取页

平台页由三栏组成：

- 左栏：我的采集表格。
  - 展示已有表格。
  - 新建表格。
  - 帮助说明。
- 中栏：当前平台扒取控制区。
  - 主页链接 / 单作品链接模式切换。
  - 链接输入框。
  - 开始扒取。
  - 登录平台。
  - 自动去重、记忆保存等状态提示。
- 右栏：采集结果。
  - 表格 / 卡片 / 详情视图切换。
  - 导出 Word / Excel / CSV。

## 采集流程

### 单作品链接

1. 用户选择平台。
2. 切换到单作品链接。
3. 一行一个链接粘贴到输入框。
4. 点击开始扒取。
5. 软件调用现有 `extract_from_page` 解析元数据。
6. 如果是视频且没有自带逐字稿，调用现有 ASR 链路转写。
7. 结果写入本地数据库。
8. 页面结果区实时刷新。

### 主页链接

1. 用户选择平台。
2. 切换到主页链接。
3. 粘贴作者主页链接。
4. 点击开始扒取。
5. 软件打开专用浏览器窗口。
6. 如果未登录，引导用户完成登录。
7. 软件读取主页可见作品列表。
8. 用户可以控制采集数量或停止。
9. 对每个作品链接复用单作品采集链路。
10. 结果保存到当前采集表格。

## 状态设计

每条结果必须有明确状态，避免用户觉得“没反应”：

- 待开始
- 登录中
- 采集中
- 元数据完成
- 转写中
- 成功
- 图文作品
- 需登录
- 平台限制
- 下载失败
- ASR失败
- 网络异常
- 待人工确认

状态旁边展示可行动错误说明，例如“请在弹出的抖音窗口完成登录后重试”。

## 技术架构

推荐第一版采用“本地 Web App + 桌面启动器”：

- 后端：复用现有 Python 采集器，新增本地 API 服务。
- 前端：本地网页界面，复刻当前橙子 mockup。
- 数据库：SQLite 保存采集表格和采集结果。
- 桌面入口：Mac `.app` 或 `.command` 启动本地服务并打开窗口。
- 采集核心：继续复用 `content_link_collector.py` 中已稳定的抖音、小红书、B站、ASR、登录浏览器逻辑。

这样可以最大限度降低风险，不重做已经跑通的平台采集链路。

## 测试重点

- 软件能打开首页。
- 平台卡片 hover 和点击正常。
- 橙子默认浮动、hover 榨汁、眨眼动画正常。
- 能新建采集表格。
- 能在不同表格之间切换。
- 单作品抖音链接能采集并保存。
- 主页链接能弹出登录/采集浏览器。
- 结果能在表格视图显示。
- 导出 CSV、Excel、Word 文件可打开。
- 失败时状态和错误说明清晰。

## 成功标准

第一版完成后，用户可以不打开飞书，直接在本地软件里完成：

1. 选择抖音。
2. 新建一个采集表格。
3. 粘贴单作品链接或主页链接。
4. 触发登录页并完成登录。
5. 自动采集作品信息和逐字稿。
6. 在软件页面查看结果。
7. 下次打开仍能看到历史记录。
8. 导出成文档或表格。
