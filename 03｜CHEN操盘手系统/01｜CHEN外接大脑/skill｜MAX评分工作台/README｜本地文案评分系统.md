# MAX 文案评分系统｜CHEN 本地版

## 系统定位

这是 CHEN BRAIN 内部长期使用的 MAX 文案评分系统。

用途：

- 新建 MAX 文案 / 脚本 / 选题评分记录
- 保留每条内容的评分历史
- 保留修改稿版本
- 对比人工评分与 AI 盲评
- 持续沉淀语感校准样本
- 可选同步到飞书多维表格

本系统是本地 Obsidian 知识库的一部分，不是独立代码项目。

## 日常启动

双击：

```text
启动MAX文案评分系统.command
```

浏览器打开：

```text
http://127.0.0.1:8765/
```

停止服务时双击：

```text
停止MAX文案评分系统.command
```

## 数据保留

本地评分记录保存在：

```text
scores_db.json
```

这个文件是系统的长期记录库。每次新建评分、提交人工分、生成 AI 盲评、提交修改稿，都会写入这个文件。

不要手动删除或覆盖 `scores_db.json`。需要备份时，可以复制整个 `skill｜MAX评分工作台` 文件夹。

## 迭代规则

允许迭代：

- `index.html`：评分工作台界面
- `max_score.py`：本地服务、评分记录、飞书同步逻辑
- `AGENTS.md`：评分执行协议
- `README｜本地文案评分系统.md`：本地使用说明
- `系统迭代日志.md`：每次系统改动记录

谨慎迭代：

- 权重
- 发布判断线
- 红线词
- AI 盲评维度

涉及评分体系版本、MAX 顶层定义、红线标准时，必须回到源文档：

```text
02｜MAX顶层思维系统/MAX内容评分体系 V2.1.md
```

并明确区分：

- 【源文档定稿】
- 【基于源文档推导】
- 【候选方案】
- 【待市场验证】
- 【暂不确定】

## 盲评协议

文案评分必须遵守：

1. 先创建记录
2. AI 先盲评
3. AI 明细在人工分提交前不展示
4. 人工评分提交后再展示 AI 分、人工分、差异、发布判断
5. 任一维度差异 >= 2 分，进入语感校准库

详细规则见：

```text
AGENTS.md
```
