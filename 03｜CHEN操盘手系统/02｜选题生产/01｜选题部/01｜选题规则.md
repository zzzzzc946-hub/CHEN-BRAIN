# 01｜选题规则

## 一、文档定位

本文件是 CHEN 操盘手系统的选题前端规则。

它不负责写稿，不负责生成最终标题，也不负责替代 MAX 判断。

它负责承接全网信息检索 Skill 抓取来的外部信息，并将其中有价值的信息整理成 MAX 可以口喷、`10-MAX内容生产 Skill` 可以承接的选题前置卡。

选题规则的本质是一个接口规则：

```text
上游：全网信息检索 Skill
中游：选题规则 / MAX选题库
下游：10-MAX内容生产 Skill
```

本文件只回答一件事：

> 外部信息怎样被筛选、整理、转化为 MAX 可以讲、内容生产 Skill 可以承接的前置结构。

---

## 二、选题规则在系统中的位置

当前选题规则处在以下链路中：

```text
全网信息检索 Skill
↓
信息筛选
↓
场景化整理
↓
结果 / 反馈 / 争议提取
↓
MAX 可讲切入点
↓
进入 MAX选题库
↓
交给 10-MAX内容生产 Skill
↓
MAX 口喷 / 正文成型
↓
反推选题和标题
```

必须明确：

1. 选题规则不是用来一开始生成标题的。
2. 选题规则不是用来直接写正文的。
3. 选题规则是中间接口，负责把外部信息转化成“可进入内容生产的前置结构”。
4. 标题和最终选题必须后置，在 MAX 口喷、正文成立后反推。
5. 前端阶段最重要的不是标题，而是：真实信息、具体场景、用户体感、反馈结果、争议、切入点。

---

## 三、上游：全网信息检索 Skill 输入标准

未来全网信息检索 Skill 提供的信息，不能只是一条热点或新闻标题。

进入选题判断前，信息至少应尽量包含：

### 1. 信息来源

平台 / 账号 / 链接 / 发布时间

### 2. 事件或现象

发生了什么，用户看见的具体东西是什么。

### 3. 讨论反馈

评论区在争什么，用户在吐槽什么，大家的情绪是什么。

### 4. 可观察结果

这个现象带来了什么结果，暴露了什么问题，形成了什么判断。

### 5. 与 MAX 用户的关系

它是否能关联到高端展示面、价值感、社交关系、情感吸引、审美判断、身份表达等方向。

### 6. 可讲性

MAX 是否能从这个信息里讲出自己的判断，而不是只复述新闻。

---

## 四、中游：信息进入选题库的筛选规则

外部信息不是因为“热”就能进入选题库。

它必须满足至少三点：

### 1. 有具体场景

不是抽象观点，不是宏大议题，不是泛泛趋势。

### 2. 有反馈 / 结果 / 争议

这个信息必须已经产生用户反应、现实结果、评论争议、身份判断或情绪反馈。

### 3. 能承载 MAX 判断

MAX 不能只是复述这个信息，而是能借这个场景讲出关于价值感、展示面、吸引力、身份表达、审美或关系结果的判断。

### 淘汰规则

以下信息不进入选题库：

1. 只有热度，没有用户体感。
2. 只有新闻事实，没有可讲场景。
3. 只有宏大趋势，没有具体人物、动作、场面或反馈。
4. 只能做资讯复述，不能承载 MAX 判断。
5. 和当前阶段用户无关。
6. 需要大量背景解释才能讲明白。
7. 只有标题刺激，但没有内容延展空间。

## 五、选题前置卡标准格式

所有通过筛选的信息，进入选题库前应整理为以下格式。

```text
【选题前置卡】

信息来源：
平台 / 账号 / 链接 / 时间

话题 / 母题：
这条信息归属于哪个大方向

具体场景 / 现象：
用户实际看见了什么，发生了什么

结果 / 反馈 / 争议：
这个现象带来了什么结果、评论、争议、误解、吐槽或身份判断

目标用户为什么会关心：
这和 MAX 当前阶段用户有什么关系

MAX 可讲切入点 1：
这个场景可以从哪个方向讲

MAX 可讲切入点 2：
这个场景可以从哪个方向讲

MAX 可讲切入点 3：
这个场景可以从哪个方向讲

推荐切入点：
当前最值得优先给 MAX 口喷的方向

可供 MAX 口喷的问题：
用 1-3 个问题激发 MAX 口喷

状态：
待判断 / 可口喷 / 已口喷 / 已成稿 / 已发布 / 已淘汰
```

### 场景与切入点的定义

硬规则：

```text
场景不是观点。
场景也不是单纯现象。
场景 = 现象 + 可观察结果 / 反馈 / 争议。

切入点不是标题。
切入点不是一句钩子。
切入点 = 从哪个方向讲这个场景，用它承载 MAX 的判断。
```

示例：

```text
错误：
场景：很多男人不会展示价值感。

正确：
场景：一个男生发了五星级酒店下午茶照片，但评论区觉得他像第一次去，反而显得刻意。

错误：
切入点：这种酒店照，女生一眼就知道你不常来。

正确：
切入点：从“高级场所不等于高级表达”切入，讲为什么一个人去了有价值的地方，不代表他就能把自己的价值表达出来。
```

## 六、下游：交给 10-MAX内容生产 Skill 的接口标准

当一条信息完成选题前置卡后，才可以交给 `10-MAX内容生产 Skill`。

交给 10 号 Skill 的不是最终标题，而是：

1. 话题 / 母题
2. 具体场景 / 现象
3. 结果 / 反馈 / 争议
4. 推荐切入点
5. 可供 MAX 口喷的问题
6. 必要素材来源

`10-MAX内容生产 Skill` 负责：

1. 承接 MAX 口喷
2. 整理正文结构
3. 打磨表达
4. 在正文成立后反推选题和标题

选题规则不替代 10 号 Skill。

选题规则只负责把外部信息加工成 10 号 Skill 可承接的输入。

---

## 七、禁止事项

1. 禁止一开始就生成最终标题。
2. 禁止把热点当选题。
3. 禁止把新闻复述当选题。
4. 禁止把抽象观点当场景。
5. 禁止把 MAX 观点提前写进场景字段。
6. 禁止把切入点写成标题。
7. 禁止为了完整而生成复杂判断大表。
8. 禁止把内容评分体系前置到选题阶段。
9. 禁止新建 `话题道场.md`。

---

## 八、入库与后置反推规则

所有通过筛选的内容，进入：

```text
03｜CHEN操盘手系统/02｜选题生产/01｜选题部/02｜MAX选题库.md
```

进入选题库时，先记录前端共创结果：

1. 信息来源
2. 话题 / 母题
3. 具体场景 / 现象
4. 结果 / 反馈 / 争议
5. 多个切入点
6. 推荐切入点
7. 状态

等 MAX 口喷、内容成型后，再补充：

1. 反推选题
2. 反推标题
3. 成稿链接 / 内容记录
4. 发布状态
5. 数据反馈
