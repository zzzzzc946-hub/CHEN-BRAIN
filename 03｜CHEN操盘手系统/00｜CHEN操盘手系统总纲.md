# 00｜CHEN操盘手系统总纲

## 一、文档定位

CHEN 操盘手系统是 CHEN BRAIN 中负责把 MAX 思想转化为内容结果的执行系统。

它不替代 `02｜MAX顶层思维系统`，也不改写 MAX 原始思想。

它负责把 MAX 的思想、表达、客户反馈、市场情报和内容数据，转化为更准的选题、更有效的内容和更稳定的发布判断。

## 二、CHEN 的操盘手角色

CHEN 是 MAX IP 项目的操盘负责人。

CHEN 负责：

- 判断 MAX 当前最应该表达什么。
- 判断客户是否真的需要、是否听得懂、是否会被击中。
- 判断选题能否进入内容生产。
- 判断文案是否值得发布。
- 判断系统建设是否偏离内容结果。

## 三、当前阶段目标

当前阶段目标不是继续搭大系统，而是：

```text
提高 MAX IP 话题 / 选题 / 内容判断准确度。
```

阶段原则：

- 先产出，再沉淀。
- 先测试，再建模。
- 先做内容，再完善 SOP。
- 先判断客户是否在意，再判断理论是否完整。

## 四、操盘手系统服务什么

操盘手系统服务四件事：

```text
客户理解
情报分析
选题生产
内容评分与反馈
```

它不追求文档数量完整，而追求判断链路清晰。

## 五、如何理解客户

理解客户不是泛泛知道用户是谁。

需要持续判断：

- 客户真实痛点是什么。
- 客户已经做过什么但没拿到结果。
- 客户为什么会对 MAX 的判断产生信任。
- 客户看到内容后是否会自检。
- 客户是否会意识到普通拍摄、普通聊天技巧、普通情感建议解决不了自己的问题。

## 六、如何服务选题和内容

选题与内容生产必须围绕：

```text
话题 → 选题 → 标题 → 判断 → 发布 → 反馈 → 迭代
```

当前选题部只保留两个活跃文档：

```text
03｜CHEN操盘手系统/02｜选题与内容生产/01｜选题部/01｜选题规则.md
03｜CHEN操盘手系统/02｜选题与内容生产/01｜选题部/02｜MAX选题库.md
```

内容评分体系统一放在：

```text
03｜CHEN操盘手系统/02｜选题与内容生产/02｜内容评分体系.md
```

## 七、与 MAX 顶层思维系统的关系

`02｜MAX顶层思维系统` 是 MAX 大脑源。

CHEN 操盘手系统只能基于它理解、转译、判断、生产内容，不得擅自改写 MAX 原始源文档。

涉及 MAX 顶层定义时，必须回到源文档，并区分：

```text
【源文档定稿】
【基于源文档推导】
【候选方案】
【待市场验证】
【暂不确定】
```

## 八、与总控台的关系

总控台负责项目方向、阶段进展、协作规则和 AI 复利机制。

操盘手系统负责把方向落到客户理解、选题、内容和评分反馈。

总控台管边界。

操盘手系统管产出。
