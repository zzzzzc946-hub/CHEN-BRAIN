# 02｜项目协作与仓库规则

## 一、文档定位

本文档是 CHEN BRAIN 的项目协作与仓库操作规则入口。

本仓库是 Obsidian Markdown 知识库，不是普通代码项目。

Codex、ChatGPT、Obsidian、GitHub 的所有协作，都必须服务：

```text
更懂 MAX → 更懂 MAX 客户群体 → 判断是否有价值、是否有效 → 产出更准的选题和内容
```

## 二、硬性规则

Codex 不得默认创建：

```text
src/
app/
config/
package.json
```

除非 CHEN 明确要求搭建代码项目。

Codex 不得主动：

- 修改 `02｜MAX顶层思维系统` 中的 MAX 原始源文档。
- 移动 `02｜MAX顶层思维系统` 中的任何文件。
- 重构 `02｜MAX顶层思维系统`。
- 主动扩系统。
- 主动 commit / push。
- 删除旧文件。

所有高风险动作先归档，不直接删除。

## 三、目录边界

```text
00｜MAX IP项目总控台
```

用于项目方向、阶段进展、协作规则、AI 复利机制。

```text
01｜Inbox｜md转换
```

用于临时导入、格式转换、原文件归档。

```text
02｜MAX顶层思维系统
```

MAX 大脑源。默认只读，不参与项目治理重构。

```text
03｜CHEN操盘手系统
```

用于客户理解、情报、选题、内容生产、评分反馈和工具执行。

## 四、文件移动、删除与归档

旧文档、重复文档、废弃文档先移动到对应：

```text
99｜历史归档
```

归档时保留原文件名，并在文件顶部注明：

```text
本文档已归档。
归档原因：已被新版合并文档替代 / 旧结构废弃 / 待确认。
新入口：对应新文档路径。
```

不确定去向的文件统一放入：

```text
03｜CHEN操盘手系统/99｜历史归档/待确认
```

或：

```text
00｜MAX IP项目总控台/99｜历史归档/原04-AI协同系统待确认
```

## 五、GitHub 与 Obsidian

GitHub 仓库是 ChatGPT 读取 CHEN BRAIN 的云端入口。

Obsidian 是本地知识库和长期沉淀入口。

Codex 可以维护仓库文件、执行迁移、做 Git 状态检查，但不得主动提交或推送。

需要提交或推送时，必须由 CHEN 明确要求。

## 六、AI 协作偏差处理

涉及历史偏差、重复错误、协作规则冲突时，参考：

```text
00｜MAX IP项目总控台/04｜AI协作与复利机制.md
```

不要因为发现偏差就自动扩系统。

只有用户明确要求，或阶段复盘确认需要沉淀时，才更新相关规则。
