本仓库是 Obsidian Markdown 知识库，不是普通代码项目。

# 搭建实录检查规则

每次 Codex 准备提交前，必须检查本次变更是否涉及以下内容：

1. 项目目录结构变化
2. 项目总控台规则变化
3. AI协同系统规则变化
4. Codex操作规则变化
5. Obsidian / GitHub / Codex 协作机制变化
6. MAX思想系统AI读取层变化
7. CHEN操盘手系统关键结构变化
8. 新增可教学、可复用、可迁移的方法论流程
9. 完成一个可被记录为“阶段”的项目搭建动作

如果涉及以上内容，Codex 必须在提交前提醒：

    本次变更可能需要更新 07｜CHEN BRAIN搭建实录与教学手册.md。

然后自动生成一段“阶段记录草稿”，包含：

1. 阶段目标
2. 操作前的问题
3. ChatGPT判断
4. CHEN最终决策
5. Codex执行动作
6. 新增 / 修改 / 删除文件
7. 最终形成的规则
8. 本阶段结论
9. 下一步动作

Codex 不得擅自重写 `07｜CHEN BRAIN搭建实录与教学手册.md` 全文。

Codex 只能：

- 在文末追加新阶段
- 不修改已有阶段记录
- 不删除历史记录
- 不重写全文
- 不改动 MAX 原始源文档

# 项目文档批量同步规则

项目文档默认采用批量同步机制，不因每次小改都更新日志或规则文档。

ChatGPT/Codex 应在以下情况主动提醒 CHEN 批量同步：

- 完成一个阶段
- 连续形成一组相关规则
- 一个系统模块收尾
- 多个文件产生关联变更
- 进入下一个模块前
- 距离上次同步已积累较多未记录决策

提醒时只给出需要同步的文档和精简Codex指令。

批量同步的默认节奏为：

    日终批量同步 + 阶段收尾同步

日常交流先累计待同步事项，不因小改频繁调用 Codex。

每天收尾、阶段完成、进入下一模块前、准备提交时，由 ChatGPT 主动提醒 CHEN 批量同步。

Codex 执行批量同步时，应一次处理多文件更新，减少小步消耗。

# 文档自动化分级规则

CHEN BRAIN 文档自动化分为三级。

A级：必须主动提醒同步。

适用对象：

    项目进展
    Codex规则
    AI协作规则
    MAX思想AI读取层

处理方式：

    ChatGPT/Codex 必须主动判断是否需要同步。
    需要同步时，提醒 CHEN 并给出精简 Codex 指令。
    Codex 执行时只追加或局部更新，不重写全文。

B级：阶段性批量同步。

适用对象：

    CHEN操盘手系统SOP
    Prompt
    工作流

处理方式：

    日常先累计。
    阶段完成、模块收尾、进入下一模块前批量同步。
    不因单次小改立即调用 Codex。

C级：不自动化。

适用对象：

    MAX原始源文档
    临时草稿
    一次性说明文档

处理方式：

    不建立自动提醒。
    不因自动化规则改写、移动、删除或重命名。
    仅在 CHEN 明确要求时处理。
