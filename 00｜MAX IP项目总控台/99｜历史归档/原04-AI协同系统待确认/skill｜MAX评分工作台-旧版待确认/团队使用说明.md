# MAX 内容评分工作台 · 团队使用说明

## 日常使用

1. 双击 `启动MAX评分工作台.command`
2. 浏览器会打开 `http://127.0.0.1:8765/`
3. 新建评分记录，粘贴文案
4. 给人工总分、最多2个主因、一句话评价
5. 提交后查看发布前总评分、总结戳和 AI 六维诊断

## 飞书配置

### 方式一：自动创建飞书表

1. 给飞书自建应用开通 `bitable:app` 或 `base:app:create`
2. 发布/启用应用
3. 双击 `创建飞书评分表.command`
4. 输入 App ID 和 App Secret
5. 脚本会创建「MAX内容评分」多维表格，写入 `config.json`，并初始化字段

### 方式二：使用已有飞书表

1. 在飞书里创建一个多维表格
2. 把自建应用添加为文档应用，并给可编辑权限
3. 双击 `配置已有飞书评分表.command`
4. 输入 App ID、App Secret、app_token、table_id
5. 脚本会自动补齐字段

## 评分逻辑

发布前总评分为10分制：

```text
发布前总评分 = 人工评分 × 70% + AI评分 × 30%
```

人工前台只给：

- 总分
- 主要问题，最多2个
- 一句话评价

AI 后台保留六维诊断，用来长期校准语感。

## 注意

- `config.json` 含 App Secret，不要发到公开仓库。
- 每台电脑本地都有自己的 `scores_db.json`，飞书表才是团队共享沉淀层。
- 如果同步失败，先运行 `python3 max_score.py sync-test` 看缺字段或权限。
