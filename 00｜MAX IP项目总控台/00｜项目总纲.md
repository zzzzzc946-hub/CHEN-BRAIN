# 00｜项目总纲

## 一、CHEN BRAIN 是什么

CHEN BRAIN 是服务 MAX IP 项目的 Obsidian Markdown 知识库与工作系统。

当前阶段，它不是 CHEN 的完整人生级第二大脑，也不是普通代码项目。

它的核心任务是帮助 CHEN 更稳定地理解 MAX、理解 MAX 的客户群体，并把这些理解转化成更准的选题、内容、评分和复盘。

## 二、当前项目主线

CHEN BRAIN 当前所有结构最终服务一条主线：

```text
更懂 MAX
↓
更懂 MAX 的客户群体
↓
判断是否产生价值、是否有效
↓
产出更准的选题和内容
```

系统存在的意义不是为了把目录越搭越完整，而是为了让 MAX IP 的内容判断越来越准。

## 三、旧模块如何收敛

早期 CHEN BRAIN 曾采用五个一级模块：

```text
00｜MAX IP项目总控台
01｜Inbox｜md转换
02｜MAX顶层思维系统
03｜CHEN操盘手系统
04｜MAX × CHEN AI协同系统
```

本次收敛后，`04｜MAX × CHEN AI协同系统` 不再作为独立一级目录保留。

AI 协作规则、复利纠错、Codex 协作机制收敛到：

```text
00｜MAX IP项目总控台
```

MAX 内容评分体系、评分工作台、内容训练相关工具收敛到：

```text
03｜CHEN操盘手系统
```

`02｜MAX顶层思维系统` 是 MAX 大脑源，不参与项目治理重构，不移动、不重构、不改写原始源文档。

## 四、当前一级目录职责

```text
00｜MAX IP项目总控台：项目方向、阶段进展、协作规则、AI复利机制
01｜Inbox｜md转换：临时导入与转换区
02｜MAX顶层思维系统：MAX大脑源，不参与项目治理重构
03｜CHEN操盘手系统：客户理解、情报、选题、内容生产、评分反馈
```

## 五、当前结构原则

```text
全面但精简
清晰但不复杂
先产出，再沉淀
先测试，再建模
先做内容，再完善 SOP
```

所有目录调整、文档合并、工具维护，都必须回到 MAX IP 内容结果导向。
