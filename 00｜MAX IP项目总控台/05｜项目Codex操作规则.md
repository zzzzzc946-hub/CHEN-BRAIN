# 05｜项目Codex操作规则

## 一、文档定位

本文档用于记录 Codex 在 `CHEN BRAIN` 仓库中的项目级操作规则。

本仓库是 Obsidian Markdown 知识库，不是普通代码项目。

Codex 执行任务时，必须遵守：

    不擅自删除文件
    不擅自移动文件
    不擅自重命名核心目录或源文档
    不擅自改写 MAX 原始思想
    不创建 src、app、package.json 等代码项目结构
    提交前检查是否需要更新搭建实录

------------------------------------------------------------------------

# 搭建实录检查规则

每次 Codex 准备提交前，必须检查本次变更是否涉及以下内容：

1. 项目目录结构变化
2. 项目总控台规则变化
3. AI协同系统规则变化
4. Codex操作规则变化
5. Obsidian / GitHub / Codex 协作机制变化
6. MAX思想系统AI读取层变化
7. CHEN操盘手系统关键结构变化
8. 新增可教学、可复用、可迁移的方法论流程
9. 完成一个可被记录为“阶段”的项目搭建动作

如果涉及以上内容，Codex 必须在提交前提醒：

    本次变更可能需要更新 07｜CHEN BRAIN搭建实录与教学手册.md。

然后自动生成一段“阶段记录草稿”，包含：

1. 阶段目标
2. 操作前的问题
3. ChatGPT判断
4. CHEN最终决策
5. Codex执行动作
6. 新增 / 修改 / 删除文件
7. 最终形成的规则
8. 本阶段结论
9. 下一步动作

Codex 不得擅自重写 `07｜CHEN BRAIN搭建实录与教学手册.md` 全文。

Codex 只能：

- 在文末追加新阶段
- 不修改已有阶段记录
- 不删除历史记录
- 不重写全文
- 不改动 MAX 原始源文档

------------------------------------------------------------------------

## 二、提交前检查清单

每次提交前，Codex 应检查：

    是否误改 MAX 原始源文档
    是否误移动或重命名一级目录
    是否误提交 .DS_Store、workspace、密钥或缓存
    是否只暂存用户指定范围
    是否需要更新搭建实录
    是否需要输出待确认事项

如果本次变更涉及项目机制建设，Codex 应优先提醒 CHEN 更新搭建实录。

------------------------------------------------------------------------

# MAX思想系统AI读取层提交前检查规则

每次 Codex 准备执行 `git add` / `git commit` 前，必须先检查本次 `git diff` / `git status` 是否涉及以下目录或文件：

```text
02｜MAX顶层思维系统/
02｜MAX顶层思维系统/00｜MAX源文档清单.md
02｜MAX顶层思维系统/00｜MAX思想系统AI读取总览.md
02｜MAX顶层思维系统/00｜MAX核心概念索引.md
02｜MAX顶层思维系统/00｜MAX源文档AI摘要总表.md
04｜MAX × CHEN AI协同系统/01｜MAX AI知识库系统.md
```

如果本次变更涉及 `02｜MAX顶层思维系统/` 下任何 MAX 源文档新增、修改、删除、版本变化或目录变化，Codex 必须在提交前提醒：

    本次变更涉及 02｜MAX顶层思维系统，可能需要执行：开始同步MAX思想系统AI读取层

Codex 应先执行同步检查，再决定是否需要更新：

    00｜MAX源文档清单.md
    00｜MAX思想系统AI读取总览.md
    00｜MAX核心概念索引.md
    00｜MAX源文档AI摘要总表.md
    04｜MAX × CHEN AI协同系统/01｜MAX AI知识库系统.md

如果同步检查发现没有新增、修改、删除或版本不一致，Codex 应在提交前说明：

    本次未发现需要同步的 MAX 源文档变化。

无论是否需要同步，Codex 都不得因为提交前检查而擅自：

    改写 MAX 原始源文档
    移动 MAX 原始源文档
    删除 MAX 原始源文档
    重命名 MAX 原始源文档
    重构 02｜MAX顶层思维系统 目录

------------------------------------------------------------------------

# AI协作偏好提交前沉淀规则

每次提交前，若本轮对话或变更中出现新的AI协作偏好、Codex硬性要求、自动化规则、输出风格要求，Codex必须主动提醒CHEN是否写入：

    04｜MAX × CHEN AI协同系统/09｜CHEN × AI协作偏好与硬性规则.md

CHEN确认后，只追加，不重写全文。

------------------------------------------------------------------------

# 自动化判断总规则

以后 CHEN BRAIN 所有系统窗口中，凡出现持续更新、容易遗忘、需要同步、需要记录、需要检查的文档或流程，ChatGPT/Codex 必须主动判断是否需要自动化。

若需要自动化，默认设计为：

    Codex主动检测 → 主动提醒CHEN → CHEN确认 → Codex执行

不得默认依赖CHEN主动想起触发。

每次提交前，如果本次变更涉及项目日志、迭代记录、阶段任务、目录命名、Codex规则、AI协作规则、SOP、Prompt、工作流，Codex必须主动提醒是否同步对应文档。

------------------------------------------------------------------------

# 项目更新日志提交前检查规则

提交前若本次变更涉及项目级决策或CHEN与ChatGPT形成的新规则，Codex必须主动提醒是否更新：

    00｜MAX IP项目总控台/03｜项目更新日志.md

如果 CHEN 确认需要更新，Codex 只能追加当日更新记录，不得重写全文，不得删除历史记录。

------------------------------------------------------------------------

# 全系统窗口通用同步判断规则

CHEN BRAIN所有系统窗口通用。

凡CHEN与ChatGPT交流中产生项目级决策、系统边界、自动化机制、Codex规则、AI协作规则、操盘方法、SOP、复盘方法或目录归位判断，ChatGPT必须主动判断应更新到哪个目录/文档，并给出精简Codex指令。

Codex提交前也必须检查本轮变更是否需要同步：

    00｜MAX IP项目总控台/03｜项目更新日志.md
    00｜MAX IP项目总控台/07｜CHEN BRAIN搭建实录与教学手册.md
    04｜MAX × CHEN AI协同系统/09｜CHEN × AI协作偏好与硬性规则.md
    对应系统文档
