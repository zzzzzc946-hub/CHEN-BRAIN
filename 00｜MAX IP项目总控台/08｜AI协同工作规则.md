# 08｜AI协同工作规则

## 一、文档定位

本文档用于定义 ChatGPT、Codex 以及其他 AI 工具在 `CHEN BRAIN` 仓库中的协同方式。

本项目中的 AI 不是 MAX，也不是 CHEN。

AI 的定位是：

    辅助理解
    辅助整理
    辅助生成
    辅助执行
    辅助复盘
    辅助提效

------------------------------------------------------------------------

## 二、AI 进入项目后的必读顺序

AI 进入本仓库后，应优先读取以下文档：

    00｜MAX IP项目总控台/00｜项目总纲.md
    00｜MAX IP项目总控台/07｜CHEN BRAIN仓库协作规则.md
    00｜MAX IP项目总控台/08｜AI协同工作规则.md
    00｜MAX IP项目总控台/09｜GitHub与Obsidian同步规则.md
    02｜MAX顶层思维系统/00｜MAX顶层思维系统说明.md
    03｜CHEN操盘手系统/00｜CHEN操盘手系统说明.md
    04｜MAX × CHEN AI协同系统/00｜MAX × CHEN AI协同系统说明.md

读取顺序的原则是：

    先理解项目总方向
    再理解仓库规则
    再理解 AI 边界
    最后进入具体系统

------------------------------------------------------------------------

## 三、ChatGPT 的主要职责

ChatGPT 更适合承担：

    战略讨论
    内容构思
    结构梳理
    Prompt 设计
    文案草稿
    复盘分析
    思想提炼

ChatGPT 输出的内容可以作为草稿、建议和思考材料，但不自动等于正式项目结论。

------------------------------------------------------------------------

## 四、Codex 的主要职责

Codex 更适合承担：

    读取仓库文件
    检查 Git 状态
    新增 Markdown 文档
    按规则修改文件
    维护 .gitignore
    辅助整理目录
    输出变更清单
    协助 Git 提交

Codex 在执行前应先确认任务边界，尤其是涉及删除、移动、重命名、核心文档改写和 Git 操作时。

------------------------------------------------------------------------

## 五、AI 可以做什么

AI 可以：

    阅读现有文档
    总结项目结构
    生成规则文档草稿
    提出目录优化建议
    协助转换文档
    协助内容生产
    协助复盘
    协助检查 Git 状态

------------------------------------------------------------------------

## 六、AI 不可以擅自做什么

AI 不可以擅自：

    删除文件
    删除 Inbox 源文件
    重命名一级目录
    移动核心系统文档
    修改 MAX 核心思想
    修改 CHEN 操盘角色定位
    改写项目总方向
    提交敏感配置到 GitHub
    把代码项目结构强加到知识库中

------------------------------------------------------------------------

## 七、AI 输出进入仓库的规则

AI 输出进入仓库前，应区分三种状态：

    草稿
    建议
    正式文档

草稿可以保留较多探索内容。

建议应说明理由、影响和可选方案。

正式文档应结构清晰、边界明确、可被后续 AI 和 CHEN 直接读取。

------------------------------------------------------------------------

## 八、最终判断权

MAX 负责思想源头。

CHEN 负责操盘判断。

AI 负责效率放大。

所有 AI 输出都必须经过 CHEN 判断后，才能进入正式项目系统。
