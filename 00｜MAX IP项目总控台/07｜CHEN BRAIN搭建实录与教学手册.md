# 07｜CHEN BRAIN搭建实录与教学手册

## 一、文档定位

本文档用于持续记录 `CHEN BRAIN` 从 0 到 1 的搭建过程。

它不是普通更新日志，而是项目搭建实录、协同教学手册和后续可复用方法论沉淀文档。

本文档记录：

    每个阶段为什么要做
    做之前遇到什么问题
    ChatGPT如何判断
    CHEN如何决策
    Codex实际执行了什么
    最终沉淀了哪些规则
    后续可以如何复用

后续 Codex 不得重写本文档全文。

Codex 只能在文末追加新阶段，不修改已有阶段记录，不删除历史记录。

## 阶段001｜确定 CHEN BRAIN V1.0 目录结构

### 1. 阶段目标

确定 `CHEN BRAIN｜V1.0` 的项目根目录结构，让 Obsidian、ChatGPT、Codex 和 GitHub 后续可以围绕同一个知识库稳定协作。

### 2. 操作前的问题

项目刚开始时，`CHEN BRAIN` 需要承担很多可能方向：

    MAX IP项目
    CHEN个人第二大脑
    AI协同系统
    内容生产系统
    未来商业化系统

如果一开始就按照“人生级第二大脑”搭建，目录会过早膨胀，反而影响当前最重要的 MAX IP 项目推进。

### 3. ChatGPT判断

ChatGPT 判断当前阶段不应该提前搭建过度复杂的全局系统。

`CHEN BRAIN｜V1.0` 应先服务于 MAX IP 项目。

先跑通 MAX IP 项目的知识库、操盘系统和 AI 协同机制，再考虑未来扩展成更完整的 CHEN BRAIN 全局系统。

### 4. CHEN最终决策

CHEN 决定：

    CHEN BRAIN V1.0 当前服务于 MAX IP 项目
    不提前搭建 CHEN 的完整人生级第二大脑
    一级目录先稳定为五个核心系统

最终一级目录为：

    CHEN BRAIN
    ├── 00｜项目总控台
    ├── 01｜Inbox｜md转换
    ├── 02｜MAX顶层思维系统
    ├── 03｜CHEN操盘手系统
    └── 04｜MAX × CHEN AI协同系统

### 5. Codex执行动作

Codex 读取当前仓库结构，确认 Obsidian 仓库根目录、Git 状态、`.obsidian` 配置和现有 Markdown 文档。

Codex 没有重构五个一级目录。

Codex 后续围绕该结构建立仓库协作规则和 AI 读取层。

### 6. 新增 / 修改 / 删除文件

本阶段核心动作是确定目录结构和项目定位。

不以新增文件为主要目标。

已形成并保留的一级目录：

    00｜MAX IP项目总控台
    01｜Inbox｜md转换
    02｜MAX顶层思维系统
    03｜CHEN操盘手系统
    04｜MAX × CHEN AI协同系统

### 7. 最终形成的规则

    目录稳定性 > 目录完美性

五个一级目录不得随意删除、移动、重命名或合并。

后续所有 ChatGPT / Codex / Obsidian / GitHub 协作，都应先尊重该 V1.0 结构。

### 8. 本阶段结论

CHEN BRAIN V1.0 当前服务于 MAX IP 项目，而不是提前搭建 CHEN 的完整人生级第二大脑。

### 9. 下一步动作

完成四个根文档，为总控台、MAX思想系统、CHEN操盘系统和 AI协同系统建立基础说明。

### 4. CHEN最终决策

## 阶段003｜建立 MAX 思想系统 AI读取层 V1.0

### 1. 阶段目标

为 `02｜MAX顶层思维系统` 建立 AI 读取层，让 ChatGPT 和 Codex 能稳定吸收 MAX 持续输入的大量源文档，并把思想内容转化到 CHEN 操盘系统和 AI 协同系统。

### 2. 操作前的问题

`02｜MAX顶层思维系统` 中已经出现大量 MAX 源文档和相关材料。

这些文档体量大、层级深、更新频率高。

如果 ChatGPT 或 Codex 每次都直接读取所有原文，会出现：

    读取成本高
    容易漏重点
    容易把源思想和AI提炼混在一起
    容易把外接大脑误当成MAX本体思想
    难以判断新增/更新/删除后该同步哪里

### 3. ChatGPT判断

ChatGPT 判断不应该重构 `02｜MAX顶层思维系统` 的原始目录。

MAX 会持续向 `02` 输入源文档，CHEN 和 ChatGPT 不负责强行改造 MAX 的输入结构。

正确做法是：

    保留 MAX 原始源文档
    不重构 02 目录
    在 02 顶部建立 AI 读取层
    在 04 中建立 AI 知识库同步规则

### 4. CHEN最终决策

CHEN 决定建立 MAX 思想系统 AI读取层 V1.0。

AI 读取层采用单文件摘要总表方案，而不是多个独立摘要卡长期并行维护。

### 5. Codex执行动作

Codex 创建并提交了 MAX 思想系统 AI读取层相关文件。

Codex 建立了源文档清单、AI读取总览、核心概念索引、源文档AI摘要总表和 AI知识库系统规则。

Codex 同时在 `04｜MAX × CHEN AI协同系统` 中补充了 MAX 思想源文档增量同步规则。

### 6. 新增 / 修改 / 删除文件

已建立文件：

    02｜MAX顶层思维系统/00｜MAX源文档清单.md
    02｜MAX顶层思维系统/00｜MAX思想系统AI读取总览.md
    02｜MAX顶层思维系统/00｜MAX核心概念索引.md
    02｜MAX顶层思维系统/00｜MAX源文档AI摘要总表.md
    04｜MAX × CHEN AI协同系统/01｜MAX AI知识库系统.md

曾短暂创建的 `AI摘要/` 文件夹和 7 个单篇摘要卡，已根据 CHEN 决策合并进：

    02｜MAX顶层思维系统/00｜MAX源文档AI摘要总表.md

并删除单篇摘要卡与空摘要文件夹，避免后续出现两套摘要来源。

### 7. 最终形成的规则

未来 MAX 更新 `02｜MAX顶层思维系统` 后，使用固定触发指令：

    开始同步MAX思想系统AI读取层

Codex 执行增量同步：

    新增源文档 → 清单新增记录 → 摘要总表追加章节 → 总览/概念索引局部更新
    更新源文档 → 清单标记更新 → 摘要总表刷新对应章节 → 总览/概念索引局部更新
    删除源文档 → 清单标记删除 → 摘要移入已失效摘要 → 总览/概念索引局部调整

增量同步必须遵守：

    只更新受影响章节
    不全量重写
    不改写 MAX 原始思想
    不重构 02｜MAX顶层思维系统 目录
    所有重大不确定项标记为待确认

### 8. 本阶段结论

由于 `02｜MAX顶层思维系统` 由 MAX 持续输入大量源文档，CHEN 和 ChatGPT 不负责重构该目录，而是建立 AI 读取层，用于稳定吸收 MAX 思想并转化到 CHEN 操盘系统。

### 9. 下一步动作

后续每次 MAX 源文档新增、更新、删除或版本不一致时，使用固定触发指令启动增量同步。

同时，在项目提交前检查是否形成新的可教学阶段，并决定是否追加到本文档。
