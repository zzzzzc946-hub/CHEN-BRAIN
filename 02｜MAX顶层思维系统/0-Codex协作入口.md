# Codex 协作入口

这个文件是 Codex 深度参与 MAX BRAIN 工作时的固定入口。

每次开始一个新任务，先判断任务属于哪一类，再按对应路径调用资料。

---

## 1. 内容输出任务

典型指令：

- 写一篇 MAX 外部内容
- 根据某个选题生成短文
- 改写成 MAX 语感
- 判断一段内容像不像 MAX

调用顺序：

1. `3-MAX外部表达系统/MAX语感校准库.md`
2. `3-MAX外部表达系统/MAX外部传播手册 V1.md`
3. `4-MAX外接大脑/00-外接大脑调用总则.md`
4. 根据选题决定是否调用：
   - `4-MAX外接大脑/01-JZ表达方式与阶层观察.md`
   - `4-MAX外接大脑/02-浩威情感逻辑观点库.md`
5. 必要时回到 `1-中枢文档/` 校准底层判断。

输出前检查：

- 有没有真实场景
- 有没有关键动作
- 有没有暴露问题
- 有没有回到 MAX 判断
- 有没有太像 AI 总结、概念翻译或模板句

---

## 2. 内容评分与质检任务

典型指令：

- 给这篇内容打分
- 判断哪里不像 MAX
- 用评分体系检查
- 生成修改建议

优先读取：

1. `MAX内容评分体系 V2.1.md`
2. `maxscore-codex/AGENTS.md`
3. `3-MAX外部表达系统/MAX语感校准库.md`
4. `5-MAX内容输出测试/内容成品库.md`

输出方式：

- 先给结论
- 再指出主要偏差
- 再给可执行修改方向
- 不要为了完整而写废话

---

## 3. 价值观系统任务

典型指令：

- 开始进行价值观梳理
- 归档这条价值观
- 更新价值观汇总
- 判断这条属于核心还是衍生

必须读取：

1. `1-中枢文档/05-MAX价值观系统/MAX价值观系统操作文档 V1.md`
2. `1-中枢文档/05-MAX价值观系统/00-价值观收集池.md`
3. `1-中枢文档/05-MAX价值观系统/价值观汇总.md`
4. `1-中枢文档/05-MAX价值观系统/01-核心价值观/`

关键规则：

- 先生成梳理报告，不直接归档。
- 用户确认后，再创建或更新正式文件。
- 废弃价值观只保留原文，不展开。

---

## 4. 系统集成与自动化任务

典型指令：

- 接入飞书
- 把 Obsidian 内容同步到飞书
- 建一个评分工具
- 批量整理语料
- 做脚本自动化

优先读取：

1. `6-系统集成/AI协同配置清单.md`
2. `6-系统集成/飞书-Codex桥接/README.md`
3. 具体工具目录下的 README 或 AGENTS 文件。

执行原则：

- 先保护密钥和原始资料。
- 自动化脚本要可重复运行。
- 默认不覆盖用户原文件，除非用户明确允许。
- 对外部 API 操作先做小规模测试。

---

## 5. 日常资料整理任务

典型指令：

- 整理这批语料
- 从逐字稿里提炼观点
- 把素材放进对应模块
- 找出可沉淀的表达

调用顺序：

1. 先读用户给出的具体素材。
2. 再读 `数字版AI MAX目录说明.md` 判断归属。
3. 如果涉及外接能力，读 `4-MAX外接大脑/00-外接大脑调用总则.md`。
4. 如果涉及对外表达，读语感校准库和传播手册。

整理原则：

- 原始语料和提炼观点分开。
- 不把外部 IP 的人设搬进 MAX。
- 只迁移能力、观察方式和判断结构。
