# 外接大脑提纯 Skill 执行规范

## 定位

本规范只用于外接大脑语料提纯。

本规范不适用于：

- MAX 自有语料提纯。
- MAX 案例库提纯。
- MAX 价值观正式归档。
- MAX 对外内容写作。

目标不是改写外接大脑，不是净化外接大脑，不是判断对错，也不是 MAX 化。

目标只有一个：

> 从外部语料中提纯出对 MAX 有帮助、未来可供 AI 写稿调用的观点弹药。

## 默认输出结构

除非用户另行指定，外接大脑提纯表只保留三列：

| 原文 | 观点提纯 | 来源 |
| -- | ---- | -- |

不得默认新增：

- 编号
- 类型
- 主题
- 备注
- 调用次数
- 风险提醒
- 可生成选题
- MAX 翻译

## 字段规则

### 原文

保留原语料中有价值的片段。

可以是一句话，也可以是一小段。

允许做轻微转写修复：

- 删除明显口癖和无意义噪音。
- 修复明显错字。
- 补齐因转写导致的不通顺断句。
- 不改变原意。

不允许为了显得高级而重写原文。

### 观点提纯

观点提纯必须是对原文核心意思的提炼，不能复制原文。

合格标准：

- 是一句完整观点。
- 忠实反映原文本来在说什么。
- 比原文更清楚、更可调用。
- 不加入 MAX 判断。
- 不加入 Codex 判断。
- 不做道德修正。
- 不把极端表达改温和。
- 不把简单话复杂化。

禁止：

- 原文复制一遍当观点提纯。
- 只删除几个字当观点提纯。
- 写成 MAX 语言。
- 写成安全、正确、高级的二次解释。

自检规则：

如果“观点提纯”和“原文”相似度过高，必须重写。

示例：

```text
原文：
你越是倾尽所有，在她眼里越是廉价，爱情也需要引领者，你不敢主导的瞬间，就是她开始动摇的瞬间。

观点提纯：
过度付出会让男人显得廉价，缺少主导感会让关系开始动摇。
```

不合格示例：

```text
观点提纯：
你越是倾尽所有，在她眼里越是廉价，爱情也需要引领者。
```

### 来源

来源格式跟随用户指定。

浩威语料默认格式：

```text
浩威语料合并外接大脑｜编号 标题
```

示例：

```text
浩威语料合并外接大脑｜006 low男只会付出，高阶男调动情绪
```

## 提纯密度

不追求少量精品，也不为了数量凑条目。

判断标准：

> 这条内容以后能不能帮助 AI 更理解情感关系、男性处境、关系结构、用户痛点，或者让 MAX 写稿更有弹药？

能，就提。

相近观点可以保留，只要它们来自不同场景、不同表达、不同用户痛点。

高度重复、只是换皮的观点，应在合并阶段去掉。

## 分段规则

大语料必须分段提纯，不能一次性全量读完后直接输出。

分段目的：

> 保证每个阶段的阅读质量、判断质量和观点提纯质量不下降。

### 先判断语料规模

开始前先统计：

- 总行数。
- 总字数或大致字符量。
- 一级/二级编号数量。
- 每篇平均长度。
- 是否存在单篇特别长的语料。

优先按编号分段。

如果没有编号，则按标题或自然段分段。

### 推荐分段数量

按“每段 40-70 篇左右”作为默认目标。

可按以下规则调整：

| 语料规模 | 建议分段 |
| -- | -- |
| 30 篇以内，且总量不大 | 1 段 |
| 31-100 篇 | 2 段 |
| 101-180 篇 | 3 段 |
| 181-260 篇 | 4 段 |
| 260 篇以上 | 5 段或更多 |

如果单篇很长，优先增加分段。

如果每篇很短，可以减少分段。

浩威语料这类 200 篇以上的大文档，默认分 5 段较合理。

## 标准流程

### 1. 先做 5 条测试样本

在全量执行前，必须先给用户 5 条测试样本。

测试样本只在对话中展示，不写入正式表。

用户确认后，再进入全量提纯。

测试样本用于校准：

- 原文保留尺度。
- 观点提纯力度。
- 转写修复尺度。
- 是否允许相近观点保留。
- 输出字段是否正确。

### 2. 分段提纯

每段单独生成一个 Markdown 文件。

文件名遵循用户指定格式。

如果用户没有指定，使用：

```text
外接大脑提纯表_来源名001-060.md
外接大脑提纯表_来源名061-120.md
```

每段只处理该段范围，不跨段。

每段完成后做一次轻检查：

- 是否只有三列。
- 来源编号是否在该段范围内。
- 是否存在明显复制原文的观点提纯。
- 是否存在明显转写乱码。

### 3. 合并

所有分段完成后，生成完整合并表。

合并时只做三件事：

1. 合并所有分段表格。
2. 删除完全重复或高度重复内容。
3. 保留原意更清楚、更适合后续调用的版本。

合并表仍然只保留三列。

不得在合并阶段新增分类、备注或判断。

### 4. 最终检查

最终必须检查：

- 分段文件是否都存在。
- 完整合并表是否存在。
- 每个表是否只有三列。
- 每段来源编号是否越界。
- 完整表总条数。
- 抽样检查观点提纯是否不是原文复读。

## 失败判定

出现以下情况，视为流程失败，必须返工：

- 观点提纯大面积复制原文。
- 输出字段多于用户要求。
- 把外接大脑改写成 MAX 语言。
- 为了安全或正确擅自修正原观点。
- 为了压缩数量删掉大量可用观点。
- 一次性全量处理超长语料导致质量明显下降。
