# 05-选题规则 V2.3

版本：V2.3｜外部情报压缩漏斗版  
定位：把外部情报日报压缩成 MAX 可口喷材料卡片

---

## 一、文档定位

本文件负责一件事：

```text
把《MAX外部情报日报》里的 KEEP / BUFFER 情报，
压缩、校准成 MAX 可读、可判断、可口喷的材料卡片。
```

它位于：

```text
04-MAX外部情报日报
↓
05-选题规则
↓
06-MAX选题库调试记录
↓
07-MAX选题库
↓
MAX口喷
↓
10-MAX内容生产 Skill
```

一句话：

```text
05 不是存情报，也不是写内容；
05 是把情报变成 MAX 能讲的材料。
```

---

## 二、上位校准标准

选题规则必须同时服从两层标准。

### 1. 长期最高标准

MAX 一直研究的是：

```text
真实价值如何形成价值感，并最终转化为吸引力和关系结果。
```

因此，一条材料是否值得进入选题漏斗，核心看它能不能帮助 MAX 解释：

```text
真实价值
价值表达
价值感形成
身份识别
审美判断
生活方式
展示面
吸引力
关系反馈
关系结果
目标用户自检
```

这些不是封闭清单，而是判断维度。

### 2. 当前阶段入口

当前阶段，MAX 用：

```text
高端展示面
```

作为可见入口，服务：

```text
有真实价值，但没有被正确看见的高价值男性。
```

所以，优先进入选题漏斗的材料，应该能帮助这个用户意识到：

```text
我不是没价值；
而是我的价值没有被正确表达、正确感知、正确转化。
```

## 三、输入 / 调试 / 输出链路

### 1. 输入：04-MAX外部情报日报

输入来自《MAX外部情报日报》里的：

```text

```

```text

```

```text

```

### 2. 处理：05-选题规则

05 负责把日报情报处理成：

```text

```

处理重点：

```text

判断用户相关性
判断 MAX 可讲性
校准切口能量
形成推荐切入点
```

### 3. 调试：06-MAX选题库调试记录

如果候选卡片出现以下问题，进入 06：

```text
方向不准
切口能量不对
太像说教
太像情感号
太负向
太抽象
不够具体
不够 JZ 式
没有提供价值
MAX 看完不想讲
CHEN / MAX 有明显纠偏
```

06 记录调试过程，并反向更新：

```text

03 日报模板
04 日报保留习惯
05 选题规则

```

### 4. 输出：07-MAX选题库

只有经过 05 压缩，并完成必要 06 调试的卡片，才能进入 07。

07 只存放：

```text

```

的材料卡片。

## 四、选题漏斗

日报情报进入 07 前，必须经过以下五步。

### 1. 材料压缩

先把日报条目压缩清楚：

```text
这条材料到底讲了什么？
它是事件、现象、故事，还是一类反馈？
它被讨论的原因是什么？
外界的反应是什么？
```

不要急着写标题。

不要急着做结论。

先把材料本身压清楚。

---

### 2. 用户相关性判断

判断它和 MAX 当前目标用户有什么关系：

```text
有真实价值但没有被正确看见的男人，会不会关心？
他会不会自检？
他会不会觉得自己也犯过？
这是否关系到展示面、身份表达、审美表达、社交表达？
这是否能帮他理解自己为什么没有被正确感知？
```

目标用户不关心，不进入 07。

---

### 3. MAX 可讲性判断

判断 MAX 能不能借这个材料讲出自己的判断。

MAX 不能只是复述资讯。

必须能讲出：

```text
为什么这个现象成立
为什么这个表达会失败
为什么这个细节会改变外界判断
为什么真实价值没有被正确感知
为什么有些人越展示越掉价
为什么有些人不刻意展示反而更有价值感
```

如果只能讲成热点评论、情感鸡汤、穿搭科普、炫富八卦、追女生技巧，而不能回到价值感主线，不进入 07。

---

### 4. 切口能量校准

判断这个材料的讲法是否符合 MAX 的表达气质。

外部情报可以来自：

```text
翻车
破绽
尴尬
误判
低位
供养
审美断层
场景不熟
```

但 MAX 的最终切口不能停留在：

```text
羞辱
批判
低位审判
穷人视角
传统情感号式压迫感
```

切口能量优先级：

```text
正向价值提供
↓
知识 / 品味 / 生活方式启发
↓
具体场景建议
↓
问题诊断
↓
刺痛 / 羞辱 / 批判
```

核心规则：

```text
抓取可以抓破绽。
日报可以记录破绽。
选题库卡片必须尽量转成正向价值、知识启发、品味判断和场景建议。
```

---

### 5. 入库判断

最后判断：

```text
这条材料是否能让 MAX 看完就想讲？
```

如果还需要解释半天背景，暂不入库。

如果 MAX 看完不知道讲什么，暂不入库。

如果 CHEN / MAX 明显感觉方向不对，进入 06 调试。

## 五、MAX 可口喷卡片格式

进入 `07-MAX选题库` 的内容，统一使用以下格式：

```text
【MAX可口喷卡片】

来源信息：
平台 / 链接 / 热度 / 来源账号

话题 / 母题：
这条材料归属的大方向。

材料摘要：
把具体事件 / 现象 / 故事 / 结果 / 反馈 / 争议合并写清楚。

目标用户为什么会关心：
这和 MAX 当前阶段用户有什么关系。

MAX 可讲切入点：
1.
2.
3.
4.
5.

推荐切入点：
当前最值得优先给 MAX 口喷的方向。
```

填写要求：

```text
材料摘要必须写具体，不写空泛观点。
切入点不是标题，不写成标题党。
推荐切入点只能选一个。
卡片只服务 MAX 口喷，不混入发布管理字段。
```

## 六、入库与淘汰规则

### 1. 可以进入 07 的材料

必须满足：

```text
有具体材料
有外部反馈或现实结果
目标用户会关心
能承载 MAX 判断
切口能量不低级
MAX 看完能直接口喷
```

### 2. 不进入 07 的材料

以下内容不入库：

```text
只有热度，没有场景
只有新闻事实，没有用户体感
只有抽象观点，没有具体动作或反馈
只能做资讯复述
和目标用户无关
情绪很大，但和价值感主线无关
能讲成泛情感、泛成功学、泛吐槽，但讲不成 MAX
AI 为了凑字段硬编出来的材料
```

### 3. 淘汰但有价值的材料

如果材料不能入库，但调试过程有价值，记录到 `06-MAX选题库调试记录`。

记录目的不是保留垃圾材料，而是沉淀：

```text
为什么它不能入库
哪里判断错了
以后如何避免
是否需要反向更新 01 / 02 / 03 / 05
```

## 七、选题与标题后置

本文件不提前生成最终选题和标题。

前端只生成：

```text

话题 / 母题
材料摘要
目标用户为什么会关心
MAX 可讲切入点
推荐切入点
```

等 MAX 口喷、内容成型后，再反推：

```text
选题
标题
发布版本
数据反馈
```

标题必须从正文里最强的部分反推：

```text
最强冲突
最强判断
最具体场景
最强用户自检
最强情绪压力
最能被目标用户点开的表达
```


交给 10 号 Skill 的核心材料：


如果没有 MAX 口喷，10 号 Skill 最多做：

```text

```

不能直接替代 MAX 输出最终发布稿。

## 九、最小输出原则

选题规则不是为了让文档看起来完整，而是为了让 MAX 更容易产出内容。

每次整理外部情报时，优先输出：

```text
少而准
可口喷
有具体材料
有反馈
有判断空间
```

不要为了数量堆卡片。

不要为了格式完整硬凑字段。

不要加入复杂评分系统。

最重要的是：

```text
每一张进入选题库的卡片，都必须提升 MAX 口喷内容的准确度。
```
