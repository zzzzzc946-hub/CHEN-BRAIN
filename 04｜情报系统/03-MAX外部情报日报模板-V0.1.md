# MAX外部情报日报模板 V0.1

## 文件定位

本文件承接：

```text
01-全网情报检索规则
02-关键词与来源池
```

它负责回答：

```text
当外部候选素材被找到并完成初步判断以后，
应该抓取哪些信息，
如何压缩成 CHEN 可审阅的情报条目，
并写入《MAX外部情报日报》。
```

本文件不是检索规则。

什么值得抓，由《01-全网情报检索规则》判断。

本文件不是关键词池。

从哪里找、怎么找，由《02-关键词与来源池》判断。

本文件不是选题库。

日报条目不等于 MAX 可口喷选题。

本文件不是内容生产 Skill。

不写口播，不成稿，不生成最终标题。

一句话：

```text
02 决定从哪里找到候选素材。
01 决定候选素材是 KEEP / BUFFER / DROP。
03 决定 KEEP / BUFFER 如何写入本地日报。
```

---

# 一、日报的作用

《MAX外部情报日报》是外部情报系统的阶段性输出。

它的作用是：

```text
把全网抓到的外部素材，
压缩成 CHEN 可以快速判断的情报条目。
```

日报只回答一个问题：

```text
外部世界发生了什么，里面有没有值得 MAX 讲的现实素材？
```

日报不是最终内容库。

日报不是选题库。

日报不是标题库。

日报不是口播稿。

日报进入下一步之前，还需要经过：

```text
选题规则
↓
MAX选题库
↓
MAX口喷
↓
10-MAX内容生产 Skill
```

---

# 二、写入日报的前提

不是所有抓到的内容都写入日报。

进入日报前，必须先经过《01-全网情报检索规则》的判断。

可以进入日报的素材包括：

```text
KEEP
BUFFER
```

不进入日报的素材：

```text
DROP
```

其中：

```text
KEEP：优先写入日报。
BUFFER：可以写入日报，但必须标注为“待观察 / 待补信息 / 需要强转译”。
DROP：不写入日报。
```

---

# 三、单条情报必须抓取的信息

一条情报写入日报前，至少要保留以下信息。

## 1. 情报标题

用一句话让 CHEN 知道这条情报是什么。

情报标题不是最终内容标题。

情报标题不是小红书标题、短视频标题或口播标题。

它只是日报内部的识别标题。

要求：

```text
短
具体
能看出事件或现象
不要写成营销标题
不要写成最终选题标题
```

示例：

```text
奢侈品品牌正在从“招待VIP”转向“进入超级VIP生活方式”
```

## 2. 事件 / 现象 / 故事梗概

把以下三件事合在一起写：

```text
发生了什么
反映了什么现象
大概故事是什么
```

不要拆太细。

不要写成理论分析。

不要急着输出 MAX 观点。

重点是让 CHEN 快速知道：

```text
这条外部素材到底在讲什么。
```

## 3. 热度与来源

尽量保留：

```text
平台
链接
来源账号
检索关键词
内容发布时间
抓取时间
播放
点赞
评论
收藏
分享
转发
热榜位置
是否正在发酵
```

没有数据就写：

```text
未提供数据
```

## 4. 评论区核心声音

有就写。

没有就空着。

优先记录：

```text
高赞评论
重复出现的评价
用户真实判断语言
争议点
嘲讽点
共识点
```

评论区核心声音不是简单摘抄。

它要帮助 CHEN 判断：

```text
外界到底在用什么语言理解这件事。
```

## 5. 初步 MAX 可讲视角

列 2-5 个切口。

这里只做方向提示。

不代表已经进入选题库。

不代表最终口喷切口。

要求：

```text
只写切口
不写完整口播
不生成最终标题
不急着判断哪个一定能讲
```

初步 MAX 可讲视角要回答：

```text
这条情报可以被 MAX 从哪些方向重新解释？
```

# 四、单条情报可选补充信息

以下信息不是必须，但能拿到就保留。

```text
封面文字
话题标签
作者简介
视频时长
完整逐字稿
本地截图
本地视频路径
同一事件的其他平台来源
是否有后续发酵
是否需要人工复查
```

这些信息主要服务后续：

```text
补事实
补评论
补语境
判断是否值得进入选题库
```

# 五、单条情报输出模板

每条日报使用以下格式。

```text
01｜情报标题

状态：
KEEP / BUFFER

事件/现象/故事梗概：
把“发生了什么 + 反映了什么现象 + 大概故事”合在一起写。
不要拆太细。

热度与来源：
平台 / 链接 / 来源账号 / 检索关键词 / 内容发布时间 / 抓取时间 / 播放 / 点赞 / 评论 / 收藏 / 分享 / 转发 / 热榜位置 / 是否在发酵。
没有数据就写“未提供数据”。

评论区核心声音：
有就写，没有就空着。

初步 MAX 可讲视角：
1.
2.
3.
```

说明：

```text
状态必须写。
KEEP 代表优先给 CHEN 判断。
BUFFER 代表有潜力，但信息不完整、需要观察或需要强转译。
```

# 六、整份日报文档格式

V0 阶段，所有情报先统一写入一个文档：

```text
MAX外部情报日报
```

暂时不按日期拆分多个 MD 文件。

每次新增情报时，按照时间顺序往下追加。

建议每一轮新增时，使用以下结构：

```text
# YYYY-MM-DD｜外部情报新增

## 本轮来源
平台 / 账号 / 热榜 / 关键词 / 用户手动提供素材

## 本轮KEEP

01｜情报标题
状态：KEEP
...

02｜情报标题
状态：KEEP
...

## 本轮BUFFER

01｜情报标题
状态：BUFFER
...

## 待补信息

- 哪些素材缺链接
- 哪些素材缺评论
- 哪些素材缺热度
- 哪些素材需要继续观察发酵
```

如果某一轮只有 1 条素材，也可以只写单条，不强制完整日报结构。

# 七、KEEP / BUFFER 在日报里的写法

## 1. KEEP

KEEP 的写法要尽量完整。

必须写：

```text
情报标题
事件/现象/故事梗概
热度与来源
初步 MAX 可讲视角
```

评论区核心声音有就写，没有可以空。

KEEP 代表：

```text
这条情报已经足够清楚，可以进入 CHEN 人工判断。
```

## 2. BUFFER

BUFFER 可以不完整。

但必须说明为什么暂存：

```text
信息不完整
评论区没抓到
热度未知
场景有价值但需要强转译
像观点说教，需要还原成具体事件
需要观察是否继续发酵
```

BUFFER 代表：

```text
这条情报有潜力，但暂时不能直接进入选题库。
```

## 3. DROP

DROP 不写入日报。

如果某类 DROP 重复出现，可以反向记录到：

```text
02-关键词与来源池
↓
低效关键词黑名单
```

# 八、日报与选题库的边界

日报解决的是：

```text
外部世界发生了什么。
```

选题库解决的是：

```text
MAX 可以讲什么。
```

所以日报条目不能直接等于选题。

日报允许：

```text
事件不够成熟
观点还没定
切口还在试
信息需要补
```

选题库不允许。

只有经过《选题规则》二次处理后，日报条目才可能进入《MAX选题库》。

执行链路如下：

```text
02 关键词与来源池：提供检索入口，找到候选素材。
↓
01 全网情报检索规则：判断 KEEP / BUFFER / DROP。
↓
03 外部情报日报模板：把 KEEP / BUFFER 写成本地日报条目。
↓
选题规则：判断是否能成为 MAX 可口喷材料。
↓
MAX选题库
```

# 九、日报禁止事项

日报不做以下事情：

```text
不生成最终标题
不写口播稿
不做内容成稿
不替 MAX 做最终表达
不把观点说教直接当选题
不把无法验证的故事当事实事件
不把热点热度当成入选理由
不把 DROP 内容写入日报
```

特别注意：

```text
无法验证但故事好的素材，可以作为典型场景。
不能作为事实事件写。
```

# 十、当前版本总则

本文件只解决一个问题：

```text
外部素材找到以后，如何存成 CHEN 可以快速审阅的日报条目。
```

核心原则：

```text
先存事实和现象。
再存热度与来源。
再存评论区声音。
最后存初步 MAX 可讲视角。
```

最重要的一句话：

```text
日报不是选题库。
日报是外部现实素材进入 MAX 内容系统前的缓冲层。
```
