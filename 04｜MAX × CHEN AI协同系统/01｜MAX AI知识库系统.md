# 01｜MAX AI知识库系统

## 一、文档定位

本文档用于规定 AI 如何读取、理解、摘要和维护 MAX IP 项目知识库。

它是 `04｜MAX × CHEN AI协同系统` 中的知识库协同规则文件。

本文档不存放 MAX 思想正文。

MAX 思想正文归入：

    02｜MAX顶层思维系统

------------------------------------------------------------------------

## 二、AI如何读取02｜MAX顶层思维系统

AI 进入 `02｜MAX顶层思维系统` 时，应按以下顺序读取：

1. `02｜MAX顶层思维系统/00｜MAX思想系统AI读取总览.md`
2. `02｜MAX顶层思维系统/00｜MAX源文档清单.md`
3. `02｜MAX顶层思维系统/00｜MAX核心概念索引.md`
4. `02｜MAX顶层思维系统/00｜MAX源文档AI摘要总表.md`
5. `02｜MAX顶层思维系统/00｜MAX顶层思维系统说明.md`
6. `02｜MAX顶层思维系统/01｜MAX顶层思维系统索引.md`
7. 具体 MAX 源文档

如果任务是生成外部内容，还应读取：

    02｜MAX顶层思维系统/03-MAX外部表达系统/MAX语感校准库.md
    02｜MAX顶层思维系统/03-MAX外部表达系统/MAX外部传播手册 V1.md

如果任务是内容创作、表达转化或 MAX IP 输出，AI 不能只读取项目上下文，还应按任务读取：

    MAX思想层
    MAX语料层
    MAX表达层
    MAX内容测试层

对应目录包括但不限于：

    02｜MAX顶层思维系统/00-中枢文档
    02｜MAX顶层思维系统/02-日常MAX语料
    02｜MAX顶层思维系统/03-MAX外部表达系统
    02｜MAX顶层思维系统/05-MAX内容输出测试

如果任务涉及外部能力参考，还应读取：

    02｜MAX顶层思维系统/04-MAX外接大脑/00-外接大脑调用总则.md

## 三、GitHub读取链路与读取边界

GitHub 作为 ChatGPT 读取 CHEN BRAIN / Obsidian 仓库的云端入口。

ChatGPT 可按任务读取仓库相关文件，但不是永久记住全库。

因此，执行 MAX 相关任务前，AI 应根据任务重新判断需要读取哪些文件。

如果是项目管理、规则沉淀或总控台协作，优先读取：

    00｜MAX IP项目总控台
    04｜MAX × CHEN AI协同系统

如果是内容创作、表达转化或外部传播，必须同时读取：

    MAX思想层
    MAX语料层
    MAX表达层
    MAX内容测试层

不能仅凭旧窗口上下文或项目总控台上下文生成内容。

------------------------------------------------------------------------

## 四、MAX AI知识库同步自动化分级

MAX 思想 AI 读取层属于 A 级自动化对象。

当出现以下情况时，ChatGPT/Codex 必须主动提醒 CHEN 是否同步：

    MAX源文档新增
    MAX源文档更新
    MAX源文档删除
    MAX源文档版本不一致
    MAX思想AI读取层规则变化

但 MAX 原始源文档属于 C 级对象。

Codex 不得因为自动化同步而擅自：

    改写 MAX 原始源文档
    移动 MAX 原始源文档
    删除 MAX 原始源文档
    重命名 MAX 原始源文档
    重构 02｜MAX顶层思维系统

自动化同步只作用于 AI 读取层、清单、摘要、索引和协同规则。

## 五、Codex如何生成摘要

所有 MAX 源文档摘要统一维护在：

    02｜MAX顶层思维系统/00｜MAX源文档AI摘要总表.md

不再使用 `AI摘要/` 文件夹作为当前摘要维护方式。

Codex 为 MAX 源文档生成摘要时，必须在摘要总表中新增或更新对应章节，并使用以下结构：

    源文档名称
    源文档路径
    文档类型
    摘要更新时间
    文档定位
    核心问题
    核心结论
    核心概念
    可转化到CHEN操盘系统的内容
    可转化到AI协同系统的内容
    不能误读的地方

摘要总表不能替代源文档。

如果摘要总表和源文档冲突，以源文档为准。

## 六、新增源文档后如何更新

当 `02｜MAX顶层思维系统` 新增 MAX 源文档后，Codex 应执行：

1. 识别新增文件。
2. 判断新文档类型。
3. 更新 `00｜MAX源文档清单.md`。
4. 在 `00｜MAX源文档AI摘要总表.md` 中新增对应摘要章节。
5. 局部更新 `00｜MAX思想系统AI读取总览.md`。
6. 局部更新 `00｜MAX核心概念索引.md`。
7. 输出变更清单，交给 CHEN 确认。

如果新文档只是原始语料、外接大脑、测试内容或临时材料，是否生成摘要应标记为 `待确认`。

### 1. 如何更新源文档清单

在 `00｜MAX源文档清单.md` 的“当前源文档清单”表格中新增一行。

必须填写：

    文件名
    路径
    文档类型
    当前状态
    是否已有AI摘要
    对应AI摘要路径
    备注

状态填写规则：

    已摘要 = 已在摘要总表中生成有效摘要章节
    待摘要 = 需要摘要但尚未摘要
    待确认 = 文档类型、归属或是否摘要不确定

如果 Codex 无法判断该文档是否属于 MAX 源文档，必须标记为 `待确认`，不能强行归类。

### 2. 如何追加摘要总表章节

在 `00｜MAX源文档AI摘要总表.md` 的“当前有效摘要”部分新增一个独立章节。

新增章节必须包含：

    源文档名称
    源文档路径
    文档类型
    摘要更新时间
    文档定位
    核心问题
    核心结论
    核心概念
    可转化到CHEN操盘系统的内容
    可转化到AI协同系统的内容
    不能误读的地方

新增摘要时，只追加新文档对应章节，不全量重写已有摘要。

### 3. 如何局部更新AI读取总览

只在必要位置补充新增源文档。

允许局部更新：

    最高优先级源文档
    各源文档一句话摘要
    AI不能误读的地方
    更新记录

如果新增文档不是最高优先级中枢源文档，不应擅自提高它的读取优先级。

### 4. 如何局部更新核心概念索引

Codex 应检查新增文档是否带来：

    新核心概念
    旧概念的新定义
    概念层级变化
    概念关系变化
    概念命名冲突

如果只是新增普通案例或表达素材，不更新核心概念索引。

如果出现新核心概念，但 CHEN 尚未确认，先在相关备注中标记 `待确认`，不要直接写入一级核心概念。

新增源文档时，只增量新增该文档相关信息，不全量重写摘要总表、读取总览或概念索引。

## 七、源文档删除后如何处理

如果源文档被 CHEN 删除、替换或明确废弃，Codex 应：

1. 不恢复源文档，除非 CHEN 明确要求。
2. 在 `00｜MAX源文档清单.md` 中将状态改为 `已删除`。
3. 将摘要总表中的对应章节移动到：

        00｜MAX源文档AI摘要总表.md 的“三、已失效摘要”

4. 在失效摘要章节顶部标注失效原因和失效日期。
5. 从 `00｜MAX思想系统AI读取总览.md` 中移除当前有效引用。
6. 检查 `00｜MAX核心概念索引.md` 是否需要移除或降级相关概念。

### 1. 如何标记源文档清单

在 `00｜MAX源文档清单.md` 中保留该文档记录，不删除该行。

将状态改为：

    已删除

对应 AI 摘要路径改为摘要总表中的“已失效摘要”章节锚点。

备注中记录：

    删除原因
    删除确认来源
    删除日期

如果删除并非 CHEN 明确确认，而只是 Codex 在文件系统中发现文件缺失，必须先标记为 `待确认`，不能直接判定为已删除。

### 2. 如何移动摘要章节

在 `00｜MAX源文档AI摘要总表.md` 中，将对应摘要章节从“当前有效摘要”移动到“三、已失效摘要”。

失效章节顶部增加：

    失效状态
    失效原因
    失效日期
    处理人

不得删除历史摘要内容。

### 3. 如何处理读取总览

如果该源文档在 `00｜MAX思想系统AI读取总览.md` 中属于当前有效引用，应从当前有效读取顺序或当前有效摘要表中移除。

如需保留历史说明，只能放入更新记录或历史备注，不得继续出现在当前有效读取路径里。

### 4. 如何处理核心概念索引

如果某个概念只来源于已删除源文档，应标记为历史来源或待确认，不应直接删除。

如果该概念仍被其他有效源文档支持，应保留概念，并移除或降级已删除文档的来源权重。

如果 Codex 无法判断概念是否仍有效，必须标记 `待确认`，等待 CHEN 或 MAX 确认。

源文档删除后，只增量移动该源文档对应摘要章节，不全量重写摘要总表。

Codex 不得因为源文档删除而自动删除历史摘要。

------------------------------------------------------------------------

## 八、源文档更新后如何刷新摘要

如果源文档发生更新，Codex 应：

1. 识别被修改的源文档。
2. 重新读取被修改的源文档。
3. 对比摘要总表中的旧摘要章节。
4. 只刷新摘要总表中的对应摘要章节。
5. 判断读取总览是否需要局部更新。
6. 判断核心概念索引是否需要局部更新。
7. 在 `00｜MAX源文档清单.md` 中将状态标记为 `已更新` 或 `已摘要`。
8. 输出本次摘要刷新说明。

### 1. 如何识别文档已更新

Codex 可通过以下方式识别：

    git status
    git diff
    文件修改时间
    CHEN明确指出某源文档已更新

如果源文档尚未进入 Git 管理，Codex 应通过文件内容和修改时间辅助判断，并在输出中说明判断依据。

### 2. 如何刷新摘要总表

只更新 `00｜MAX源文档AI摘要总表.md` 中该源文档对应章节。

允许更新：

    摘要更新时间
    文档定位
    核心问题
    核心结论
    核心概念
    可转化到CHEN操盘系统的内容
    可转化到AI协同系统的内容
    不能误读的地方

不得因为一个源文档更新而重写其他源文档摘要。

### 3. 如何判断读取总览是否需要更新

只有当源文档更新影响以下内容时，才局部更新 `00｜MAX思想系统AI读取总览.md`：

    最高优先级源文档列表
    MAX思想主链路
    源文档一句话摘要
    AI不能误读的地方
    与CHEN操盘系统或AI协同系统的边界

如果只是补充案例、表达、局部说明，不更新读取总览。

### 4. 如何判断核心概念索引是否需要更新

只有当源文档更新影响以下内容时，才局部更新 `00｜MAX核心概念索引.md`：

    新增核心概念
    删除核心概念
    改变概念定义
    改变概念层级
    改变概念之间的关系
    改变概念所属源文档

如果只是增加例子、语气修正或段落顺序调整，不更新核心概念索引。

源文档更新后，只增量刷新该源文档对应章节，不全量重写摘要总表。

如果更新涉及 MAX 核心定义、核心链路或核心价值观变化，必须标记为 `待确认`，等待 CHEN 审核。

------------------------------------------------------------------------

## 九、源文档版本不一致如何处理

如果 Codex 发现源文档存在版本不一致，例如：

    文件名写 V1
    正文标题写 V2.0

或：

    源文档清单中的版本
    摘要总表中的版本
    正文标题中的版本
    互相不一致

Codex 应执行：

1. 不擅自重命名源文档。
2. 不擅自修改源文档标题。
3. 在 `00｜MAX源文档清单.md` 的备注中标记版本不一致。
4. 在摘要总表对应章节中加入版本备注。
5. 如版本差异影响读取优先级或思想判断，在 `00｜MAX思想系统AI读取总览.md` 中标记 `待确认`。
6. 输出待确认事项，等待 CHEN 或 MAX 确认。

版本不一致不等于源文档错误。

只有 CHEN 或 MAX 确认后，才能重命名、改标题或调整版本记录。

## 十、增量同步总原则

MAX 思想源文档增量同步必须遵守：

    只更新受影响章节
    不全量重写
    不改写 MAX 原始思想
    不重构 02｜MAX顶层思维系统 目录
    不移动 MAX 原始源文档
    不删除 MAX 原始源文档
    不擅自重命名 MAX 原始源文档
    所有重大不确定项标记为待确认

Codex 可以维护：

    00｜MAX源文档清单.md
    00｜MAX源文档AI摘要总表.md
    00｜MAX思想系统AI读取总览.md
    00｜MAX核心概念索引.md
    04｜MAX × CHEN AI协同系统/01｜MAX AI知识库系统.md

Codex 不可以因为同步 AI 读取层，而直接改写 MAX 源文档正文。

## 十一、固定触发指令

当 CHEN 输入：

    开始同步MAX思想系统AI读取层

Codex 应执行以下流程：

1. 读取项目总纲、仓库协作规则和本文档。
2. 检查 `02｜MAX顶层思维系统` 中的源文档变化。
3. 对照 `00｜MAX源文档清单.md`，识别新增、更新、删除、版本不一致和待确认文件。
4. 对新增源文档，按“新增源文档后如何更新”流程处理。
5. 对已更新源文档，按“源文档更新后如何刷新摘要”流程处理。
6. 对已删除源文档，按“源文档删除后如何处理”流程处理。
7. 对版本不一致文档，按“源文档版本不一致如何处理”流程标记待确认。
8. 局部更新摘要总表、源文档清单、读取总览和核心概念索引。
9. 不修改、移动、删除、重命名任何 MAX 原始源文档。
10. 输出同步报告。

同步报告必须包括：

    新增源文档
    已更新源文档
    已删除源文档
    已刷新摘要章节
    已更新概念
    待确认事项
    未处理原因
    git status

如果没有发现变化，Codex 应输出“本次未发现需要同步的 MAX 源文档变化”，并附上检查范围。

------------------------------------------------------------------------

## 十二、AI不能做什么

AI 不得：

    重构 MAX 源文档目录
    擅自移动 MAX 原始源文档
    擅自重命名 MAX 原始源文档
    擅自删除 MAX 原始源文档
    擅自改写 MAX 核心思想
    把 AI 理解提炼冒充 MAX 原始思想
    把外接大脑内容混入 MAX 本体思想
    把外部表达语感当成理论源头
    把执行 SOP 写入 MAX 中枢源文档

------------------------------------------------------------------------

## 十三、AI输出确认规则

AI 输出分为：

    草稿
    摘要
    索引
    建议
    正式文档

所有 AI 输出都必须经过 CHEN 确认后，才能进入正式系统。

如果 AI 输出只是摘要或索引，也必须明确标注其来源和边界。

------------------------------------------------------------------------

## 十四、与其他系统的关系

`02｜MAX顶层思维系统` 负责提供思想源头。

`03｜CHEN操盘手系统` 负责将思想转为内容、账号、增长、复盘和商业化执行。

`04｜MAX × CHEN AI协同系统` 负责规定 AI 如何读取、摘要、生成、维护和复盘。

AI 不得混淆三者边界。
