# README · 飞书接入指南（一次性配置，约15分钟）

目标：让 `max_score.py sync` 能把评分记录写进飞书多维表格，实现团队共享与长期沉淀。

---

## 第一步：创建飞书自建应用（拿凭证）

1. 打开 https://open.feishu.cn/app → 「创建企业自建应用」，名称随意（如"MAX评分系统"）。
2. 进入应用 → 左侧「权限管理」→ 搜索并开通：
   - `bitable:app`（查看、评论、编辑和管理多维表格）
3. 左侧「版本管理与发布」→ 创建版本并发布（企业自建应用通常自动可用，若提示审核按指引走）。
4. 记下「凭证与基础信息」页的 **App ID** 和 **App Secret**。

## 第二步：创建多维表格并授权给应用

1. 在飞书里新建一个多维表格 Base，命名「MAX内容评分」。
2. 从浏览器地址栏取两个值：
   ```
   https://xxx.feishu.cn/base/{app_token}?table={table_id}&view=...
                           ^^^^^^^^^^^^        ^^^^^^^^^^
   ```
   `app_token` 是 base/ 后那串，`table_id` 是 table= 后那串（tbl开头）。
3. **关键一步**：表格右上角「···」→「更多」→「添加文档应用」→ 搜索你刚建的应用 → 添加并给「可编辑」权限。漏掉这步会报 403。

## 第三步：建字段（名称必须完全一致）

在该数据表中创建以下字段（默认的"文本"字段可以改名复用）：

| 字段名 | 类型 |
|---|---|
| 标题 | 文本 |
| 记录ID | 文本 |
| 内容类型 | 单选（流量型/人设型/转化型） |
| 稿次 | 数字 |
| 评价 | 单选（好内容/可以拍/需要修改/PASS） |
| 发布前总评分 | 数字 |
| 人工总分 | 数字 |
| AI总分 | 数字 |
| 人工主因 | 文本 |
| 人工评价 | 文本 |
| 人工-第一印象 / 人工-真实感 / 人工-文章深度 / 人工-场景度 / 人工-MAX贴合度 / 人工-用户价值 | 数字 ×6 |
| AI-第一印象 / AI-真实感 / AI-文章深度 / AI-场景度 / AI-MAX贴合度 / AI-用户价值 | 数字 ×6 |
| 差值告警 | 文本 |
| 修改说明 | 文本 |
| AI最弱维度 | 文本 |
| AI拿不准 | 文本 |
| 文案 | 文本 |
| 评分时间 | 日期 |

> 也可以让codex用飞书字段API自动创建，但手动建一次只要几分钟，且单选选项顺手就配好了。

## 第四步：填配置文件

复制 `config.example.json` 为 `config.json`，填入四个值：

```json
{
  "feishu": {
    "app_id": "cli_xxxxxxxx",
    "app_secret": "xxxxxxxxxxxx",
    "app_token": "bascnXXXXXXXX",
    "table_id": "tblXXXXXXXX"
  }
}
```

注意：`config.json` 含密钥，如果这个目录要进git仓库，把它加进 `.gitignore`。

## 第五步：验证（codex执行的具体命令）

```bash
# 1. 测试连接 + 校验字段是否齐全（会列出缺哪个字段）
python max_score.py sync-test

# 2. 跑一条完整流程做端到端验证
echo "这是一条测试文案。" > test.txt
python max_score.py new --title "联调测试" --type 流量型 --script-file test.txt
# 假设返回 id=260611150000，codex按AGENTS.md盲评生成 ai_260611150000.json
python max_score.py ai 260611150000 --file ai_260611150000.json
python max_score.py human 260611150000 --scores 6,6,6,6,6,6
python max_score.py sync 260611150000

# 3. 去飞书表里确认出现了这条记录，确认后删掉测试行即可
```

跑通之后，日常评分按 AGENTS.md 的盲评协议走，每条评完 `sync` 一次，数据就持续沉淀在飞书里，团队所有人在表里可见、可筛选、可做周复盘视图。

## 常见报错对照

| 现象 | 原因 | 解法 |
|---|---|---|
| 获取token失败 code=10003 | app_id/secret错 | 重新核对凭证页 |
| HTTP 403 / code=91403 | 应用没被加为表格协作者 | 回到第二步第3点 |
| 同步时报字段不存在 | 字段名不一致 | 运行 sync-test 看缺哪个，注意全半角和横杠 |
| FieldNameNotFound 单选值 | 单选字段没配对应选项 | 给"内容类型""评价"补全选项 |
