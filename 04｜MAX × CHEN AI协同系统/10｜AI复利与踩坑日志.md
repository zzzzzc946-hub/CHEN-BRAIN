# 10｜AI复利与踩坑日志

## 一、文档定位

本文档是 CHEN BRAIN 的 AI 复利纠错系统。

它不是新的项目工作台。

不是新的总控台。

不是新的 SOP 中心。

不是为了继续扩展系统。

它只解决一个问题：

> ChatGPT / Codex / AI 在协作过程中反复出现的偏差，必须被沉淀下来，避免下次继续犯同样的错误。

CHEN BRAIN 的核心主线不是管理 AI。

CHEN BRAIN 的核心主线是：

> 更懂 MAX\
> 更懂 MAX 的客户群体\
> 判断什么内容有价值、有效\
> 最终产出更准的选题和内容

所以，本日志只记录那些会影响以上主线的 AI 偏差与有效经验。

------------------------------------------------------------------------

## 二、本日志记录什么

只记录四类内容：

### 1. 重复性偏差

AI 已经犯过、用户已经纠正过、以后不能再犯的错误。

例如：

- 把选题部重新扩成复杂多文档结构
- 把 ChatGPT 项目源理解成项目治理源
- 一遇到问题就建议 Codex 操作
- 把 CHEN BRAIN 理解成“更懂 MAX”即可，而忽略“更懂客户群体”才是关键

### 2. 结构性偏差

不是一句话错了，而是判断路径错了。

例如：

- 没有区分 GitHub / Obsidian / ChatGPT项目源 / Codex执行区
- 没有区分 MAX 大脑材料和项目治理材料
- 没有区分选题生长模型和项目管理系统
- 没有区分“像 MAX”与“对 MAX 客户有用”

### 3. 高价值复用经验

AI 做对了、以后应该复用的做法。

例如：

- 输出前先判断：是否符合 MAX、是否击中客户、是否有价值、是否能产出内容
- 分析外接大脑时，不模仿人设，只提炼场景颗粒度、判断深度和表达材料
- 用户纠偏后，先提炼规则，不急着让 Codex 改文档

### 4. 项目协作硬规则更新

用户明确修正后的长期规则。

例如：

- ChatGPT 项目源只服务 MAX 大脑，不放总控台和 Codex 治理文档
- Codex 规则文件主要留在 GitHub / Obsidian 中
- 选题部当前保持轻量双文档结构，不再扩系统

------------------------------------------------------------------------

## 三、本日志不记录什么

以下内容不进入本日志：

- 普通闲聊
- 临时想法
- 没有长期复用价值的小改动
- 单条文案的具体润色意见
- 单次选题讨论中的非结构性判断
- MAX 原始思想内容
- 客户案例原始语料
- 外接大脑逐字稿

这些内容应该分别进入对应位置：

- MAX 原始思想 → `02｜MAX顶层思维系统`
- 选题结果 → 选题部现有文档
- 内容评分反馈 → MAX内容评分体系 / 内容反馈记录
- 外接大脑 → 外接大脑素材区
- 项目阶段记录 → 总控台进展记录

------------------------------------------------------------------------

## 四、记录格式

每次记录使用以下格式：

    ## YYYY-MM-DD｜编号｜偏差 / 复用点标题

    ### 类型
    偏差 / 复用经验 / 结构性规则 / 待观察

    ### 发生场景
    这次问题是在什么任务中出现的。

    ### 原表现
    AI / Codex 当时怎么做、怎么判断、怎么输出。

    ### 用户纠正
    用户如何纠正。

    ### 正确规则
    以后应该怎么判断。

    ### 适用范围
    ChatGPT / Codex / 项目源 / 选题系统 / 内容生产 / MAX大脑

    ### 后续动作
    不处理 / 进入待批量同步 / 需要更新具体文档 / 需要人工确认

------------------------------------------------------------------------

## 五、当前已确认的初始记录

## 2026-06｜001｜选题部不再扩成多文档结构

### 类型

结构性偏差

### 发生场景

AI 在总结待批量同步内容时，继续沿用了此前“选题部 8 文档结构”的旧方案。

### 原表现

AI 建议继续维护：

- 选题部说明与防偏规则
- MAX选题生长模型
- MAX母题与内容母题候选池
- MAX话题树
- MAX选题库
- 优先测试选题池
- 选题评分与筛选规则
- 选题复盘与迭代记录

### 用户纠正

用户指出：选题部已经改成只剩两个文档，相关判断已经在选题生长模型中说明，不需要这么多文档。

### 正确规则

选题部当前采用轻量结构。

不得默认恢复旧的多文件结构。

除非用户明确要求扩展，否则不建议新增选题部文档。

选题部的核心目标不是搭系统，而是让选题更准、更有效、更能产出内容。

### 适用范围

ChatGPT / Codex / 选题系统 / 项目结构判断

### 后续动作

进入长期规则。

------------------------------------------------------------------------

## 2026-06｜002｜ChatGPT 项目源不是项目治理源

### 类型

结构性偏差

### 发生场景

AI 曾建议将总控台、Codex规则、AI协作规则等文档放入 ChatGPT 项目源。

### 原表现

AI 把 ChatGPT 项目源理解成完整项目启动包，推荐放入：

- 总控台
- AGENTS.md
- Codex操作规则
- AI协作偏好
- 项目进展记录

### 用户纠正

用户指出：这些主要是给 Codex 和项目治理看的。ChatGPT 项目源应该主要用于不断完善 MAX 大脑，从而更准确生产选题和内容。

### 正确规则

ChatGPT 项目源不追求项目管理完整性。

ChatGPT 项目源只追求 MAX 大脑有效性。

优先放入：

- MAX 中枢思想
- MAX 阶段作战纲领
- MAX 产品系统
- MAX 外部传播手册
- MAX 语感校准库
- MAX 内容评分体系
- JZ / 浩威外接大脑
- JZ / 浩威数据
- 日常 MAX 语料

总控台、Codex规则、AI协作治理文档留在 GitHub / Obsidian 中，不默认进入 ChatGPT 项目源。

### 适用范围

ChatGPT项目源 / MAX大脑 / 内容生产

### 后续动作

进入长期规则。

## 2026-06｜003｜CHEN BRAIN 不是只为了更懂 MAX

### 类型

核心定位修正

### 发生场景

AI 曾将 CHEN BRAIN 的第一用途表述为“训练 AI 更懂 MAX”。

### 原表现

AI 认为：

> CHEN BRAIN 的第一用途是让 AI 更懂 MAX。

### 用户纠正

用户指出：

> 更懂 MAX 是前提，更懂 MAX 的客户群体才是关键，一切以是否产生价值、是否有效为终点，最终产出选题和内容。

### 正确规则

CHEN BRAIN 的正确链路是：

> 更懂 MAX\
> ↓\
> 更懂 MAX 的客户群体\
> ↓\
> 判断是否产生价值、是否有效\
> ↓\
> 产出更准的选题和内容

AI 判断任何材料、机制、选题、文案时，都不能只问“像不像 MAX”。

还必须问：

- 是否击中 MAX 客户？
- 客户看完是否觉得有用？
- 是否产生价值？
- 是否有内容效果？
- 是否能转化成更好的选题和内容？

### 适用范围

ChatGPT / MAX大脑 / 选题 / 内容生产 / 项目判断

### 后续动作

进入最高优先级判断链路。

------------------------------------------------------------------------

## 2026-06｜004｜不要一遇到问题就让 Codex 操作

### 类型

协作偏差

### 发生场景

用户给出语料、判断或纠偏后，AI 容易直接输出 Codex 指令。

### 原表现

AI 过早把问题转化为：

- 更新某文档
- 创建某结构
- 让 Codex 执行
- 同步到仓库

### 用户纠正

用户多次强调当前不需要 Codex 操作，重点是分析、吸收、迭代判断。

### 正确规则

用户给出语料或纠偏时，AI 默认先做三件事：

1.  提炼它对 MAX 大脑有什么增量
2.  判断它如何影响客户理解、选题和内容
3.  转化成下一轮内容生产规则

只有当用户明确说：

- 同步到仓库
- 给我 Codex 指令
- 整理成文档
- 执行批量同步

才输出 Codex 指令。

### 适用范围

ChatGPT / Codex / 项目同步 / 内容生产

### 后续动作

进入长期协作规则。

------------------------------------------------------------------------

## 2026-06｜005｜外接大脑不是模仿对象

### 类型

复用经验

### 发生场景

讨论 JZ、浩威、一条、心理学内容等外接大脑如何进入 MAX 系统。

### 原表现

外接大脑容易被误解为：

- 学 JZ 的语气
- 学浩威的人设
- 学别人的内容结构
- 把 MAX 变成竞品账号

### 用户方向

外接大脑是为了解决 MAX 当前表达深度、场景颗粒度和智慧材料不足的问题。

### 正确规则

外接大脑的作用是补充：

- 场景颗粒度
- 判断深度
- 表达材料
- 情感场景
- 审美场景
- 生活方式判断
- 客户痛点理解

但最终必须被 MAX 吃掉，转化为 MAX 的判断和表达。

外接大脑不进入 MAX 本体价值观。

### 适用范围

MAX大脑 / 外接大脑 / 内容生产 / 选题生成

### 后续动作

进入长期规则。

## 六、使用方式

### ChatGPT 使用方式

每次用户纠正 AI 后，ChatGPT 应该判断：

    这只是单次修改？
    还是结构性偏差？
    是否以后容易重复犯？
    是否会影响 MAX 客户理解、选题和内容？

如果是结构性偏差，则建议进入本日志。

### Codex 使用方式

Codex 不需要每次主动改本日志。

只有在以下情况更新：

- 用户明确要求同步
- 阶段复盘确认需要沉淀
- 多次出现同类偏差
- 某条规则影响后续长期执行

### 每日复盘使用方式

每日复盘时检查：

    今天有没有新的 AI 偏差？
    有没有新的复用经验？
    有没有需要沉淀的客户理解？
    有没有影响选题和内容的长期规则？

有则进入待批量同步。

------------------------------------------------------------------------

## 七、最高原则

本日志不是为了让系统更复杂。

本日志是为了让系统少犯重复错误。

所有记录必须服务于：

> 更懂 MAX\
> 更懂 MAX 客户\
> 判断什么有价值、什么有效\
> 产出更准的选题和内容

如果一条记录不能服务这个目标，就不要写进来。
