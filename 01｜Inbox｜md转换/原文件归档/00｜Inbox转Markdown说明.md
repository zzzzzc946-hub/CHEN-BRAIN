# 00｜Inbox转Markdown说明

## 一、为什么之前没有自动转换

Obsidian 本身不会把 `.xlsx`、`.docx`、`.pdf` 等文件自动转换成 `.md`。

`01｜Inbox｜md转换` 是一个收件箱目录，不是自动转换插件。

以后需要转换时，CHEN 只需要把文件放进：

    01｜Inbox｜md转换

然后告诉 Codex：

    开始转换Inbox

Codex 会批量转换可支持格式。

------------------------------------------------------------------------

## 二、转换原则

转换结果输出到：

    01｜Inbox｜md转换/已转换

原文件默认保留在原位置，不移动、不删除。

如果 CHEN 明确要求归档，Codex 再把原文件移动到：

    01｜Inbox｜md转换/原文件归档

------------------------------------------------------------------------

## 三、当前支持格式

当前可转换为 Markdown 的格式：

    .docx
    .xlsx
    .xlsm
    .csv
    .tsv
    .txt
    .rtf
    .pdf

说明：

- `.docx` 会转换为标题、段落和表格。
- `.xlsx` / `.xlsm` 会按 sheet 转为 Markdown 表格。
- `.csv` / `.tsv` 会转为 Markdown 表格。
- `.txt` 会保留为纯文本 Markdown。
- `.rtf` 会尽量提取纯文本。
- `.pdf` 会尽量抽取文本，但复杂排版、扫描版、图片型 PDF 可能需要人工校对。

暂不稳定支持：

    .xls
    .ppt
    .pptx
    .numbers

这些格式如需转换，建议先另存为 `.xlsx`、`.docx` 或 `.pdf` 后再放入 Inbox。

------------------------------------------------------------------------

## 四、固定触发指令

当 CHEN 输入：

    开始转换Inbox

Codex 执行：

1. 扫描 `01｜Inbox｜md转换` 根目录。
2. 跳过 `已转换`、`原文件归档`、隐藏文件和 `.md` 说明文件。
3. 将支持格式转换为 `.md`。
4. 输出转换清单、未处理文件和 git status。
5. 不移动、不删除原文件，除非 CHEN 明确要求。

------------------------------------------------------------------------

## 五、防偏规则

转换只做格式转换，不做内容判断。

如果转换内容涉及 MAX 顶层定义、MAX 原始源文档或关键思想，后续归位前必须再判断：

    【源文档定稿】
    【基于源文档推导】
    【候选方案】
    【待市场验证】
    【暂不确定】
